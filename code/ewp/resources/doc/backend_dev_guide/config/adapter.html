adapter 编写

1、adapter是配置网路请求适配器，主要分为两层配置：adapter,procedure

0.1 adapter

示例如下

{adapter,[{name,"ebankhost"},
          {host,"192.168.241.3"},
          {port,4003},
          {protocol,http},
          {return_type,xml},
          {res_convertor,xml2term},
          ]}.

主要配置项目网络请求的地址、端口、协议、报文返回类型，一般项目有几个关联服务就配置几个，每一个adapter根据需要配置多个procedure，来配置接口请求配置。

  • name：adapter的名字，调用的id
  • host：配置网络请求的ip地址
  • port：配置网路请求的端口
  • protocol：网络请求的协议：目前支持http、socket、rpc、扩展协议（extend），其中扩展协议用法如下：
 {protocol,{extend,{modulename,protocol_function}}}

extend_http(#adapter{host = Host,
                        port = Port,
                        protocol = Protocol
                       },
               #procedure{id = {Adapter, Procedure},
                          path = Path,
                          gen_log = Bool
                         },
               Params, Options,Extra) ->
    HttpOptions = proplists:get_value(http_options, Options, []),
    Options = proplists:get_value(options, Options, []),
    Url =
        case Path of
            "/" ++ _ ->
                lists:concat(["http://", Host, ":", Port, Path]);
            _ ->
                lists:concat(["http://", Host, ":", Port, "/", Path])
        end,
    PostBody = gen_req_body_from_list(Params),
    ContentType = "application/x-www-form-urlencoded;charset=utf-8",
    Post = {Url, [], ContentType, list_to_binary(PostBody)},
    Request =  {post, Post, HttpOptions, Options},
    case ewp_http_client:request(Request) of
        {ok, 200, _Headers, Res} ->
            Res;
        Other ->
            ?ewp_msg("http invoker response : ~p~n", [Other]),
            throw({error, invalid_http_response})
    end.
%%上送参数拼装成json
gen_req_body_from_list(Params) ->
    TupleList = [{"RSP_HEAD",[{"setp","1"}]},{"RSP_BODY",Params}],
    JsonObj = ?decode_tuplelist(TupleList),
    Json = ewp_json:encode(JsonObj),
    Json.

  • return_type：返回报文类型,目前支持xml、json

  • res_convertor：对于返回报文的处理，此配置与return_type联合使用。目前支持配置：none | xml2term | xml2json | {Module, Function}

注意：此参数配置在ewp.conf中配置默认值，具体使用请参照下面章节中的实例

下面是关于定制处理返回报文的方式

 {res_convertor,{modulename,functionname}}

convert_json(ReturnType,Res) ->
  %% do some check format with ReturnType

  %% convert json to term
  {ok,Obj,_} = ewp_json:decode(Res),
  Tuple = ?encode_tuplelist(Obj),
  Tuple.

0.2 2、procedure

实例如下：

{procedure, [{id, "login"},
       {adapter, "ebankhost"},
       {return_type, xml},
       {path, "mobileBank/ewp.do"},
       {gen_log, false},
       {use_sample_data,false},
       {data_sample, "public/xmlData/MB0201.xml"},
       {parameters, [{"tranCode","MB0201"}]}]
     }.

主要配置单个接口请求的配置，每一个procedure必须隶属一个adapter，其中与adapter中相同的配置，优先级要高。

  • id：procedure的调用id
  • adapter：procedure隶属的adapter
  • return_type:同adapter
  • path：请求接口的路径
  • gen_log：是否记录请求日志的开关
  • use_sample_data：模拟请求结果报文的开关，总开关在ewp.conf中 如下：
    {adapter_use_sample_data,true}.

只有当ewp.conf开关和当前开关同时打开时才会起效

  • data_sample：模拟请求结果报文的路径
  • parameters：请求接口的默认参数，当参数为{"key1",true}时表示请求时对key1进行非空校验
