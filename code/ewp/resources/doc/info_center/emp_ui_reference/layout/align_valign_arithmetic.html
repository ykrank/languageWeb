对齐算法

  • 功能说明
  • 算法说明

为了达到更好的显示效果,针对同一行元素在处理垂直剧中时,采用中心点对齐的方案. 目前支持对齐功能的控件包括td,div.

1 功能说明

通过align,valign属性可以指定元素的对齐方式. 此对齐方式将会作用到所有子元素.可以支持多行的水平和垂直对齐.

  • 对于指定了top,bottom样式的元素,垂直对齐对其无效.
  • 对于指定了left,right样式的元素,水平对齐对其无效.

之前的版本中,水平对齐是将整个内容区域整体做对齐,而不会针对每个 block 单独做对齐操作.在EMP5.3中,不再采用之前的算法.而是与垂直对齐一样的方式,针对每个block需要单独的进行水平对齐操作. 在此种情况下出现的不兼容问题,可以通过指定 left 的方式,或者将这些元素包裹在另一个 div 中的方式达到之前的效果. 垂直对齐算法采用行内对齐的方式,具体算法如下.

2 算法说明

构思方案: 根据布局计算中的自动换行操作,将同一行元素构成一个block.这样针对多行的情况,会产生多个block.block大小与内部元素实际大小为准.每个block代表一行.

  • 水平/垂直对齐时,首先将block做对齐操作.
  • 针对垂直对齐,再将每个block中的元素做对齐操作.
    • 垂直居上,block中元素上边界居上对齐.
    • 垂直剧中,block中元素中心点对齐.
    • 垂直居下,block中元素下边界对齐.

关于block大小的计算:

  • 通常取同一行元素所占的矩形区域作为block的frame(frame表示矩形区域的x,y坐标及宽高).
  • 当本行中某个元素指定了top,bottom样式后,在计算frame的高度时,忽略此元素.
  • 同上,当本行中某元素包含left,right样式后,在计算frame的宽度时,忽略此元素.

注:关于水平对齐,为了保持与原来效果一致.暂时将所有的block作为整体处理.而不对每个block单独做水平对齐操作.(5.3 算法中删除此需求).

Note:

  • 在td中默认的显示效果为水平居左,垂直居中. 而我们正常的布局算法是从左到右,自上而下. 因此当td指定水平对齐方式为居左或不指定时,td的水平对齐可以不处理.
  • 在div中默认的对齐效果为水平居左,垂直居上,与我们正常的布局效果一致.因此在div中,如果没有指明与默认效果不同的对齐方式,水平,垂直对齐可以不处理.
