1 getStyleByName

  • Description: 获取指定的样式值,此处的样式值指的是控件的实际值,而不是样式表中的值.在某些特殊情况下,如 td,tr 的高度,由于 td,tr 的 height 值有相互的作用关系,因此样式中指定的值与实际控件的高度可能会有差异.可以获取的样式参见表1.
    对于left、right、top、bottom、width、height等和布局相关的位置样式来说，我们规定该接口返回的值是实际值根据缩放比换算后的样式值。例如：
    在320480分辨率的手机上，一个样式width=100的div内有个样式width=50的label，label的left=30。那通过该接口获取left返回30，虽然未设置right样式，但依旧可以获取right为100-50-30=20。若此时通过set接口设置right=10，那通过该接口获取right=10，获取left=100-50-10=40。
    如果在640     960分辨率的手机上，虽然实际显示像素变为了2倍，但缩放后的样式依旧以320 * 480为基准，因此上述过程返回的值不变。
  • Syntax:
    control:getStyleByName(name)
  • Parameters:
    1. name(required)
      • 类型: String.
      • 说明: 样式名称 支持样式范围详见：setStyleByName
    2. control
      • 操作的控件对象,通常为通过getElements函数获取的.
  • Return:
    • 类型: String.
    • 说明: 样式值（如果涉及到像素值则返回值带有'px'）
  • Examples:
    local input = document:getElementsByName("button");
local width = input[1]:getStyleByName("width");
print(width); -- 200px

2 setStyleByName

  • Description: 设置指定样式的值,可以设置的样式参见表1.
    在容器类控件样式中对字体样式(fontsize)设置只支持写在报文中，不支持脚本动态设置 设置top,left,right,bottom,width,height时,value值必须带有单位1px.
  • Syntax:
    control:setStyleByName(name, value)
  • Parameters:
    1. name(required)
      • 类型: String.
      • 说明: 样式名称
    2. value(required)
      • 类型: String.
      • 说明: 样式值
  • Return: 无

3 getPropertyByName

  • Parameters:

5 getMatrix

  • Description: 获取当前控件的矩阵，以lua table形式返回矩阵各字段值.默认为单元矩阵 目前仅支持二维矩阵. 关于矩阵详细信息参见EMP Lua Library中关于 Matrix一章. 矩阵格式如下: struct CATransform3D
    {
    float m11, m12, m13;
    float m21, m22, m23;
    float m31, m32, m33;
    };
    返回的table中将以m11,m12等为字段名,返回它们分别对应的值.
  • Syntax:
    map:getMatrix()
  • Parameters: 无
  • Return:
    • 类型: Table.
    • 说明: 3*3矩阵{m11, m12, … m33}
  • See:
    setMatrix().

6 setMatrix

  • Description: 设置当前控件的矩阵.通过对矩阵的修改,达到图片变换的目的.

  • Syntax:

    map:setMatrix(matrix)
  • Parameters:
    1. matrix(required)
      • 类型: Table.
      • 说明: 3*3矩阵。{m11, m12, … m33}
  • Return: 无
  • See:
    getMatrix()

7 isSameNode()

  • Description:
    判断两个指定的控件是否是相同。如果两个控件是页面dom树中的同一个节点，则相同。
  • Syntax:
      control1:isSameNode(control2)
  • Parameters:
    1. control2(required)
      • 类型: Userdata
      • 说明: 待比较的节点。
  • Return:
    • 类型: Boolean
    • 说明: 同一个节点返回true，否则返回false。若传入的control2为nil，返回false。

  • Examples:

      local control1 = document:getElementById("aa");
  local control2 = document:getElementsByName("test");
  if control1 and control2 then
      local res = control1:isSameNode(control2[1]);
      if res then
          ...
      else
          ...
      end
  end

8 reload()

  • Description:
    重新刷新当前控件
    reload并不会重新解析xhtml，同时通过lua脚本修改后的样式也不会被还原。控件会应用最终的样式并重新布局自己及子控件。
    注：reload只用于刷新当前控件及其子控件。如果刷新后的布局影响到了其他同级控件，则不予考虑。
  • Syntax:
    metatable:reload(isInit)
  • Parameters:
    1. isInit(optional)_
      • 类型: Boolean
      • 说明: 是否初始化页面上控件的值，为true表示当前控件及其子控件中，能被用户修改的值都恢复为页面初始默认值。为false表示不需要恢复。默认为false。
        能被用户修改的控件包括radio、checkbox、segment、switch、inputtext和select。
  • Return:
    无

  • Examples:

      local control1 = document:getElementById("aa");
  control1:reload();

9 setInnerHTML

  • Description: 更新当前控件内容.当页面中的部分内容包含较大的数据量或需要较长的网络请求,为了提交用户体验,通常会将部分内容优先展示给用户,而将剩余内容通过局部刷新的方式显示.setInnerHTML就是一个完成这种功能的函数,它可以在页面显示后,通过更新单个控件内容的方式实现局部刷新.（只能是容器控件div, form, table, body）
    注：该功能是指替换容器控件的子控件，虽然报文里会将容器控件也写进去，但容器控件本身不需要修改属性样式等。
  • Syntax:
    control:setInnerHTML(content, optionparams)

  • Parameters:

    1. content(required)
      • 类型: String
      • 说明: 报文内容（也可以包含样式、脚本，包含样式则在根节点加style，脚本则加script）。
        报文内容可以带有样式定义也可不带，但是要求报文的父节点 必须与要更新的控件节点名称相同。
        Content的内容必须以作为根节点。
    2. optionparams(optional)
      • 类型: Table
      • 说明: optionparams包含以下可选参数.
        1. callback: 回调函数，在setInnerHTML界面完全加载结束后调用。<br/>
 注：此回调函数定义时必须有且仅有一个参数，此参数类型为Table，如果params参数不为空则在其中加入resultCode[String]，resultBuf[String]两个字段，如果为空则只有这两个字段。<br/>
 resultCode的value的类型为int类型，其值的意义如下:<br/>
 0  代表没有错误，已经加载完成。<br/>
 -1 代表html的解析出现问题。<br/>
 -2 代表lua的脚本加载出现问题。<br/>
 -3 代表本地资源出现安全问题（包括离线资源等）<br/>
 -4 代表网络加载的资源出现问题。<br/>
 目前只枚举这些错误代码，一旦出现负数就代表错误。<br/>
 目前客户端只支持0（成功）或者任何负数（错误）就可以了, 以上枚举错误码保留以备以后使用。<br/>
 resultBuf的返回对应的错误代码的可读性错误提示。方便提示模板开发和使用用户。如果没有错误，resultBuf为空字符串。
2. params: table类型参数，作为callback回调的参数使用

  • Return: 无

  • Examples: 更新界面上已有的Div控件，则指定的更新报文必须为：
    <content>
<style>
...
</style>
<script>
...
</script>
<div>
    ...
</div>
</content>

12 submit

  • Description: 表单提交。与submit按钮的提交功能相同.本函数会将指定表单中需要提交的元素内容以post方式提交到服务器.该请求默认通过加密信道加密,并自动处理服务器返回数据. 如果服务器返回页面报文,则将该报文加入缓存,并显示页面内容.如果返回数据没有页面内容(即没有body),则该报文不加人缓存,但依然执行报文中的脚本.此方法只能由form控件调用，如果当前form表单中包含密码输入框,则会将密码进行一次一密加密.
  • Syntax:
    control:submit()
  • Parameters: 无
  • Return: 无

13 appendChild

  • Description:
    将指定的控件节点添加到当前控件集合的最后一个子节点后（此方法仅对可包含子节点的控件起作用）。 tr不支持此操作.table中要求每一行的td个数相同,因此不能对单个tr增加子节点.

        // 在文档中添加事件句柄

control:removeOnClickListener(myFunction);

注意: 如果要移除事件句柄setOnClickListener() 的执行函数必须使用外部函数，如上实例所示 (myFunction)。 匿名函数，类似 "removeOnClickListener(function(){ myScript });" 该事件是无法移除的。

  • Syntax:
    removeOnClickListener(callback)

  • Parameters:

    1. callback (required)_
      • 类型: Function.
      • 说明: 需要被移除的监听函数.

  • Return: 无

  • Examples:

      function btnClick(btn, name, value)
  end

  local btn = Document:getEventElementsByName("cal");
  btn[0]:setOnClickListener(btnClick);
btn[0]:removeOnClickListener(btnClick);
