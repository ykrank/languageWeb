1 概述

默认情况下，EWP 使用 error_logger 工具记录日志，其在高并发条件下性能不够理想，因此在生产环境会影响系统性能。使用系统的 syslog 服务可以有效的降低日志记录造成的性能影响。本文将介绍 EWP 使用 syslog 服务记录日志的具体操作。

2 配置步骤

2.1 启动 syslog 模式

EWP 如果使用的源码，执行 configure 脚本时需添加 --enable-syslog 选项。如果使用 rpm 包，需要打包人员开启 syslog 模式进行打包。

2.2 syslog 配置

syslog 服务会根据 syslog.conf 配置文件内容来记录日志，该文件的存储路径为 /etc/syslog.conf。

例如，

修改 /etc/syslog.conf 配置文件，加上 user.* -/var/log/temp 这句，即把 user 类型服务生成的所有优先级的日志消息都记录到 /var/log/temp 文件中。

注： 修改完 syslog.conf 配置文件后，需要重启 syslog 服务来重载配置。重启指令为： /etc/rc.d/init.d/syslog restart

消息类型和优先级可以为'*'，表示所有类型和优先级。

3 日志记录

3.1 EWP 消息类型与优先级

EWP 使用的 syslog 消息类型是 user。三个日志输出宏对应的优先级为：

宏 优先级
ewp_log/2 debug
ewp_msg/2 info
ewp_err/2 err

3.2 优先级扩展

如果 EWP 使用的默认优先级无法满足需求，可以使用 syslog_drv 模块提供的 Api 来定制优先级。syslog_drv 支持的优先级包括：emerg、alert、crit、err、warning、notice、info 和 debug。

3.3 示例

使用 syslog_drv:debug/1 和 syslog_drv:debug/2 将 "hello world" 信息输出到 /var/log/ewp-syslog-demo 中:

首先，打开终端进入 /etc 目录下的 syslog.conf 文件，在该配置加上 user.* /var/log/mylog；

然后，在终端输入 /etc/rc.d/init.d/syslog restart ，屏幕上出现：

Shutting down system logger: [OK]
Starting system logger: [OK]
Starting kernel logger: [OK]

表示对 syslog.conf 的修改生效；

最后，如下调用：

表示 syslog 生成日志成功。

4 syslog 日志转储配置说明

4.1 Logrotate 简介

系统工作到了一定时间后，日志文件的内容随着时间和访问量的增加而越来越多，日志文件也越来越大。而且当日志文件超过系统控制范围时候，还会对系统性能造成影响。转储方式可以设为每年转储、每月转储、每周转储、达到一定大小转储。在 Linux 系统，经常使用 "logrotate" 工具进行日志转储，结合 cron 计划任务，可以实现日志文件的转储。

logrotate 程序是一个日志文件管理工具。用来把旧的日志文件删除，并创建新的日志文件，我们把它叫做 "转储"。我们可以根据日志文件的大小，也可以根据其天数来转储，这个过程一般通过 cron 程序来执行。logrotate 程序还可以用于压缩日志文件，以及发送日志到指定的 E-mail 。 /etc/logrotate.conf 为其默认配置文件指定每个日志文件的默认规则，即缺省配置。

4.2 Logrotate举例详解

生成转储文件

将所有类型错误级别为 info 的日志转储到 /var/log/test.log 日志文件中，并设置

/var/log/test.log 达到 5KB 后进行转储，转储10次，转储时压缩，转储后重启 syslog 服务。

  1. 修改 /etc/syslog.conf 文件使得如下：

     [root@tracvm /]# tail -1 /etc/syslog.conf      

 *.info /var/log/test.log

  2. 重启 syslog 服务： 

      [root@tracvm /]# /sbin/service syslog restart
  Shutting down kernel logger: [OK]
  Shutting down system logger: [OK]
  Starting system logger: [OK]
  Starting kernel logger: [OK]

  3. 创建 /etc/logrotate.d/test.log 日志转储参数配置文件，添加如下： 

      /var/log/test.log {
     rotate 10
     size = 5k
     compress
     postrotate
     killall -HUP syslog
     endscript
  }

    postrotate 指定转储后的动作。
    killall -HUP syslog 用来重新初始化系统日志守护程序 syslogd。
    最后的 endscript 用于结束 postrotate 部分的脚本。
    另外，此处配置需注意：
    尽管花括号的开头可以和其他文本放在同一行上，但是结尾的花括号必须单独成行。

  4. 配置生效的命令：

    [root@tracvm /]# logrotate /etc/logrotate.conf

  5. 查看转储后的文件 

      [root@tracvm /]# ls var/log/test.log*
  var/log/test.log var/log/test.log.1 var/log/test.log.2

4.3 cron 定时程序

要启动 cron 服务，使用 /sbin/service crond start 命令。

要停止该服务，使用 /sbin/service crond stop 命令。

要使用 cron 服务，你必须安装了 vixie-cron RPM 软件包，而且必须在运行 crond 服务。要判定该软件包是否已安装，使用 rpm -q vixie-cron 命令。

要判定该服务是否在运行，使用 /sbin/service crond status 命令。

cron 启动后，它会读取它的所有配置文件（全局性配置文件 /etc/crontab，以及每个用户的计划任务配置文件），然后 cron 会根据命令和执行时间来按时调用工作任务。

logrotate 没有守护进程，而是通过 crontab，脚本的路径： /etc/cron.daily/logrotate

查看文件 /etc/cron.daily/logrotate 确保如下：

#!/bin/sh
/usr/sbin/logrotate /etc/logrotate.conf
EXITVALUE=$?
if [$EXITVALUE != 0]; then
    /usr/bin/logger -t logrotate "ALERT exited abnormally with [$EXITVALUE]"
fi
exit 0

cron 的主配置文件是 /etc/crontab，它包括下面几行：

SHELL=/bin/bash
PATH=/sbin:/bin:/usr/sbin:/usr/bin
MAILTO=root
HOME=/

# run-parts
01 * * * * root run-parts /etc/cron.hourly
02 4 * * * root run-parts /etc/cron.daily
22 4 * * 0 root run-parts /etc/cron.weekly
42 4 1 * * root run-parts /etc/cron.monthly

前四行是用来配置 cron 任务运行环境的变量。SHELL 变量的值告诉系统要使用哪个 shell 环境（在这个例子里是 bash shell；PATH 变量定义用来执行命令的路径。cron 任务的输出被邮寄给 MAILTO 变量定义的用户名。如果 MAILTO 变量被定义为空白字符串（MAILTO=""），电子邮件就不会被寄出。HOME 变量可以用来设置在执行命令或脚本时使用的主目录。

/etc/crontab 文件中的每一行都代表一项任务，它的格式是：

minute hour day month dayofweek command

详细解释可以参考： http://blog.csdn.net/iiisan/article/details/1180419
