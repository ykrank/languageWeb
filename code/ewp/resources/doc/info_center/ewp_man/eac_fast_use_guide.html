1 概述

在EWP 5.3.7的版本中，优化了EWP管理平台，将管理平台从EWP中分离出来，并将其分成两部分：EAC（页面控制台）和ADMIN（服务插件）。由于变更内容较多，所以单独写一篇快速使用文档，来介绍如何配置无扩展功能的管理平台。
PS. 如果想使用扩展功能和参数详解，请参考《EWP管理平台使用文档》

管理平台可以管理多个后台App，这样就需要每个后台App都开放一个可供管理平台访问的端口，将这个端口号配置到管理平台的配置文件(ewp_admin/config/eac.conf)中，就可以管理该App了。

2 后台App部署

2.1 EWP版本

EWP版本需求： 5.3.7及以后

2.2 配置

下面以ebank后台App举例如何配置。

2.2.1 ewp.conf

管理后台的功能需要EWP的ADMIN插件服务，所以，需要在相应的ebank的ewp.conf配置文件中追加配置选项。
文件路径：ebank/config/ewp.conf （APP：ebank、emas、unit-test……）
参考样例(无扩展功能最简化配置)： 

...
{ewp_addon, [
　　{admin,[]}
]}.

2.2.2 yaws.conf

为管理平台开放一个可以访问的端口，在ebank的yaws.conf中需要添配置如下。
文件路径：ebank/config/yaws.conf
参考样例： 

<server admin>
    port = 4000
    listen = 0.0.0.0
    docroot = public/admin
    appmods = </admin, admin_controller></, error_url>
    auth_skip_docroot = true
    <opaque>
        admin = true
    </opaque>
</server>

server的名字为 admin
port :监听的端口（这里的端口同时也用在eac.conf的URL配置中）。
docroot :server使用的根目录，一般用来指定页面所存放的位置。
appmods :yaws回调模块。
auth_skip_docroot :是否跳过寻找docroot文件夹的.yaws_auth文件
opaque/admin = true : 管理平台使用的标识。
在项目中，端口号可以自定义配置，其余配置项可以不改。

2.3 启动

根据项目需求配置好以后，就可以启动ebank了。
启动的方法按照项目的流程即可。

3 管理平台EAC部署

EAC的部署方法与其他APP（ebank、emas）的部署方法相似，操作步骤为：

  1. 获取代码
  2. 编译
  3. 配置
  4. 启动

3.1 获取代码

将代码获取到本地：
EAC的代码路径：https://dev.rytong.me:9998/hg/proj/emp/ewp_admin
提示：请先确认是否有代码权限，若没有则可以向各组项目负责人申请。

3.2 编译

从服务器上获取EAC的代码后，放到本地

  1. 在ewp_admin路径下，执行./configure，如需要调试log，则可加上debug指令./configure --with-debug
  2. make

3.3 配置

EAC需要配置的文件：yaws.conf、eac.conf、admin_menu.conf。

3.3.1 yaws.conf

文件路径： ewp_admin/config/yaws.conf
参考样例：

<server eac>
    port = 4445
    listen = 0.0.0.0
    docroot = /var/www/apps/ewp_admin/public/admin
    appmods = </admin, eac_controller></, error_url exclude_paths />
    start_mod = ewp
    errormod_404 = error_url

    <opaque>
        eac = true
    </opaque>
</server>

需要关注的两个配置项：
port:监听的端口。
docroot:管理平台的页面资源根目录。使用默认配置即可。
端口号可自定义配置，在ewp_admin/config/yaws.conf下配置的端口是用来访问管理后台的，如http://localhost:4445/admin/login.html

3.3.2 eac.conf

管理后台访问后台APP的地址配置，只有正确的配置才能登陆到指定的后台APP。
文件路径： ewp_admin/config/eac.conf

配置方式： 

{app_urls, [{AppName, AppUrl}, 
            {AppAlias, AppName, AppUrl}]
}.

参考样例： 

{app_urls, [{"emas", "192.168.64.127:4443"},
            {"ebank", "192.168.64.127:4000"},
            {"unit-test", "ebank", "192.168.64.127:4009"}]
}.

AppName：应用名，参数类型：string()
AppAlias：应用别名，如"unit-test"，使用别名"unit-test"登录，但是实际使用的应用名为"ebank"，参数类型：string()
AppUrl：访问后台App的AppUrl，参数类型：string()
特别解释：AppUrl -> "IP:Port"
IP：后台App的服务器IP地址
Port：后台App开放的端口号，参考访问ebank yaws.conf中的port配置

3.4 启动

在ewp_admin路径下，执行脚本./iewp。
根据启动管理后台服务器的IP地址和ewp_admin/config/yaws.conf配置的端口号：4445，访问管理后台http://localhost:4445/admin/login.html，帐号、密码均为admin，选择需要管理的AppName，点击登陆。就进入需要管理的后台App的管理后台了。
