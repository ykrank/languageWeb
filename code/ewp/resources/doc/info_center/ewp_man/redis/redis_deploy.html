Redis操作说明

1 安装

    tar xzf redis-X.tar.gz #解压
    cd redis-X #进入代码目录
    make PREFIX=xxxx install #安装到xxxx目录

执行redis-server启动服务，执行redis-cli访问服务。

[root@localhost redis]# redis-server --daemonize yes
[root@localhost redis]# redis-cli
127.0.0.1:6379> set demo "hello world"
OK
127.0.0.1:6379> get demo
"hello world"

1.1 环境设置

  • overcommit memory系统参数设置为1：在/etc/sysctl.conf中修改vm.overcommit_memory = 1，然后重启。或者执行命令sysctl vm.overcommit_memory=1
  • 禁用内核transparent huge pages特性：echo never > /sys/kernel/mm/transparent_hugepage/enabled
  • 设置Redis的maxmemory配置。这样当内存占用过多时，Redis会报错，但不会出现故障。
  • Redis被用于频繁写的应用时，保存RDB文件或重写AOF日志可能会消耗2倍内存。
  • 使用主从复制时，保证主节点使用持久化，或者保证主节点不会自动重启。从节点会复制主节点数据，如果主节点重启且没有任何数据存储时，从节点数据也会被清空

1.2 配置

配置示例

2 Redis集群部署

Redis集群操作需要使用源码中src目录的redis-trib.rb脚本，需要先安装ruby并下载redis的gem文件。下载完成后执行gem install -l redis-3.0.5.gem。

注：ruby版本需要1.8.7及以上版本，gem需要2.5.1及以上版本。

2.1 使用脚本创建

可以使用Redis源码中utils/create-cluster目录下的create-cluster脚本在本地创建一个集群。

  • 进入utils/create-cluster/目录
  • 执行./create-cluster start
  • 执行./create-cluster create

该脚本默认会创建三主三从6个节点，占用30001端口到30006端口。使用create-cluster stop可以停止服务。

2.2 手动创建

Redis集群需要至少三个主节点，每个主节点至少有一个从节点。因此初始化一个集群需要至少6个节点。为方便说明，示例中将所有节点都部署在同一台服务器中。

创建以下配置文件，命名为redis-30001.conf。

## 服务端口
port 30001

## 集群开关
cluster-enabled yes

## 集群配置文件
## 该文件为节点记录集群状态的文件，不需手动维护
cluster-config-file nodes-30001.conf

## 节点超时时间，单位为毫秒。用于集群节点检测其他节点可用性的超时时间。
cluster-node-timeout 2000

## 开启aof日志
appendonly yes

## aof日志文件，用于持久化数据
appendfilename appendonly-30001.aof

## rdb日志文件，用于持久化数据
dbfilename dump-30001.rdb

## 节点日志文件
logfile 30001.log

## 后台启动服务开关
daemonize yes

使用以下命令启动节点。

redis-server redis-30001.conf

重复上述步骤再启动5个节点，端口号分别为30002、30003、30004、30005和30006。启动5个节点。

使用redis-trib.rb脚本执行以下命令创建集群：

./redis-trib.rb create --replicas 1 127.0.0.1:30001 127.0.0.1:30002 127.0.0.1:30003 127.0.0.1:30004 127.0.0.1:30005 127.0.0.1:30006

create为创建集群的指令，--replicas参数表示从节点数量，命令最后为节点列表。该命令会自动选择主节点和从节点，并分配哈希槽。执行后会看到以下信息。

2.3 重新分片

重新分片就是移动哈希槽的操作，在添加或删除主节点时，通常需要重新分片来使得各主节点均匀地持有的哈希槽。

使用以下指令可以执行分片操作：

    ./redis-trib.rb reshard 127.0.0.1:30001

执行后会出现的第一个提问是需要移动的哈希槽个数：

    How many slots do you want to move (from 1 to 16384)?

第二个问题是移动的目的地：

    What is the receiving node ID?

此时需要获取目的节点的ID，可通过以下指令查询：

$ redis-cli -p 30001 cluster nodes |grep myself
4602cbb9e89b62c5cc3ea19b8202c7d90c6966fc 127.0.0.1:30001 myself,master - 0 0 7 connected 0-5460

其中第一个值4602cbb9e89b62c5cc3ea19b8202c7d90c6966fc就是ID

CLUSTER NODES指令输出的信息为：

    * 节点ID
    * ip:port
    * 标记：master, slave, myself, fail, ...
    * 如果是从节点，显示主节点ID
    * 上一个PING等待回复的时间
    * 上一个PONG接收的时间
    * epoch配置
    * 节点连接状态
    * 包含的哈希槽

第三个问题是源节点ID，输入all会选择所有节点移动。如果需要指定节点，则一次输入一个ID，最后输入done表示完成。

第四个问题是分片计划的确认，输入yes后可看到哈希槽移动的日志。

使用以下指令可以检查重新分片后的集群健康情况：

    redis-trib.rb check 127.0.0.1:30001

使用以下指令可以通过参数直接执行分片，而避免交互模式参数输入：

    ./redis-trib.rb reshard --from  --to  --slots  --yes :

2.4 添加节点

2.4.1 添加主节点

无论是添加master节点还是slave节点，首先都需要创建一个空节点。

创建一个Redis配置文件 redis-30007.conf并添加以下内容：

port 30007

cluster-enabled yes

cluster-config-file nodes-30007.conf

cluster-node-timeout 2000

appendonly yes

appendfilename appendonly-30007.aof

dbfilename dump-30007.rdb

logfile 30007.log

daemonize yes

使用该配置启动新节点

    redis-server redis-30007.conf

使用以下指令将新节点添加到集群中：

# redis-trib.rb add-node 127.0.0.1:30007 127.0.0.1:30001

使用以下指令查看集群信息，检查节点是否添加成功：

# redis-cli -p 30007 cluster nodes

与其他master相比，新增的空节点有两个特点：

  • 因为没有哈希槽，所以不会存储任何数据。
  • 故障恢复不会生效

现在可以通过之前的示例向该节点分配哈希槽了。

    redis-trib.rb reshard --from 4602cbb9e89b62c5cc3ea19b8202c7d90c6966fc --to 23cf90cadf132058a360e2b65af0a61aebbbaa11 --slots 1000 --yes 127.0.0.1:30001

2.4.2 添加从节点

用同样方法创建一个使用30008端口的空节点。然后执行以下指令：

#  redis-trib.rb add-node --slave 127.0.0.1:30008 127.0.0.1:30001

与添加master不同，添加从节点的指令会使用--slave选项。redis-trib脚本会将新增节点随机添加给从节点较少的master节点。

此外，添加节点时还可以指定目标master，指令如下：

    redis-trib.rb add-node --slave --master-id 3c3a0c74aae0b56170ccb03a76b60cfe7dc1912e 127.0.0.1:7006 127.0.0.1:7000

首先添加一个空master节点，再使用CLUSTER REPLICATE指令将其转为从节点是另一种手动指定master的方法，此方法还可以用来将一个已添加的slave节点转成其他master的从节点。该指令需要在被转移的从节点上执行。

2.5 删除节点

通过以下指令可以删除节点：

    redis-trib.rb del-node 127.0.0.1:30001 ``

第一个参数是集群中任意的节点，第二个参数是被删除的节点ID。 从节点可以被直接删除，但在删除主节点前，必须保证主节点是一个空节点。因此，需要将待删除的master节点的哈希槽全部分配给其他master节点。
