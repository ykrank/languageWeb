系统可用性规范

  • 概述
    • 业务处理能力
    • EWP限流措施

概述

通过以下规范约束保证平台上发布的应用具有隔离性，不会影响平台上运行的的其他业务。

1 业务处理能力

  1. 平台上发布应用所访问的后台服务接口(一般由JUMP平台开发提供)， 其平均事务处理能力应>1000 TPS,平均响应时间应<0.5s;
  2. 应用的开发方式应当遵循EMP插件规范，将应用所需的静态界面资源作为EMP离线资源缓存到客户端本地加载， 避免在网络中重复传输;

2 EWP限流措施

根据业务级别，在EWP层对不同类型业务对应的后台服务接口最大并发数量和最长响应时间进行配置， 当EWP服务中所处理的同类业务到达设定的阙值时，拒绝后续的此类业务请求，以避免请求堵塞 占用过多的系统资源。
