系统安全规范

  • 概述

概述

与以往的系统架构不同，3.0手机银行系统将作为移动应用开发平台接受来自于总行、分行和第三方开发团队开发的应用， 手机银行系统将会面临上述所有应用所带来的安全风险。因此，从系统安全出发，我们对于平台上的应用开发有如下 规范约束。

说明：这里的应用开发指对同一个客户端APP上的功能进行扩展，例如分行开发人员在手机银行中添加水电缴费业务，它对应为EMP定义的一个业务功能插件。 开发人员可按照业务需求开发一个和多个业务插件，通过EMP提供的开发工具打包为一个应用，然后按照下文描述的流程发布到手机银行中。

1 应用开发规范

  1. 所有的应用开发应当基于EMP提供的客户端SDK和开发工具，将应用前端代码封装为符合EMP插件规范的应用包；
  2. 应用开发者须使用EMP提供的网络通讯API，以及EMP提供的前端开发框架和模板库；
  3. 应用首次开发时，须参考融易通提供的《生产环境安全检查参考手册》中客户端部分的相关要求，如：对于敏感信息输入应使用安全键盘；

2 应用管理规范

应用经过开发者测试完成之后，统一提交到行内测试平台进入应用审核流程，审核内容包括:

  1. 应用源码是否遵循EMP插件规范，源码中是否包含对非法API的调用；
  2. 应用流程是否畅通，界面风格和用户体验是否满足统一要求；
  3. 应用是否符合《生产环境安全检查参考手册》相关要求；

经过审核之后的应用，方可发布到手机银行生产环境。

3 应用运行规范

  1. 基于手机银行APP运行的所有应用，其运行时的统一入口为EWP服务，不允许APP直接访问其他的后台业务系统（包括获取APP上的界面）；
  2. APP和EWP之间的通讯方式统一使用EMP产品自定义的TLS实现，推荐使用国密算法，并打开Android的防篡改校验；
  3. 对于运行时的可访问API设置必要的鉴权机制，对信任级别较低的应用开发者（如第三方开发者）设置较低的API权限级别， 并进行运行时的API鉴权检查；
