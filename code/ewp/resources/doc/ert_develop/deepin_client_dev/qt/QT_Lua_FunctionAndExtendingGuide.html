QT Lua 功能及扩展说明

1 背景

客户端界面要针对lua脚本做界面元素的响应，如界面跳转，控件移位，按钮响应，界面元素背景换图换色操作，悬浮界面等。随着公司产品的升级换代，个性化设计等，lua接口会有更新，补充的要求。

2 需求说明

针对客户端界面的特殊设计，以及后台模版的新需求，对lua接口及功能进行扩展

Lua功能及模块扩展

1 扩展lua基本功能

扩展的基本功能是不需要与主程序交互的。如某类数学运算，计算字符长度，获取系统信息，字符串加密，编解码等。

1.1 放在LuaUtility类中的基本功能

第一步，在类中添加功能函数的静态声明和定义。

eg：static int trim(lua_State *lua)

第二步，增加数组的个数luaL_Reg utility_libs[9]; 9变为10。

第三步，在构造函数中向数组添加相应功能函数

（eg： 

reg.name = "新加功能别名（需统一定义）";

reg.func = 新加功能函数名（可自定义）;

utility\_libs[8] = reg;

reg.name = NULL;

reg.func = NULL;

utility\_libs[9] = reg;

）

第四步，脚本中使用通过luaL_register(lua, "utility", utility_libs); 函数注册的库名。 utility：新加功能别名。

1.2 放在新建类中的基本功能

第一步，创建类（类名可自定义，可参考LuaUtility创建）

成员变量： luaL_Reg 库名_libs[n+1];

（n为功能函数个数，1为NULL）

成员变量： static 类名 *pinstance

成员函数：static LuaUtility *Instance();

成员函数：int luaopen_库名libs(lua_State *lua);

第二步，放在新建类中的基本功能

（同放在LuaUtility类中的基本功能的步骤）

第三步，在LuaManage：：initLua()函数中调用新建类相关方法。

（eg：LuaUtility::Instance()->luaopen_utilitylibs(lua_s);）

第四步，脚本中使用成员函数（luaopen_库名libs）中通过luaL_register(lua, "新库名", utility_libs); 函数注册的库名 新库名：新加功能别名

2 扩展lua与主程序的交互功能

要扩展API，要了解程序的驱动逻辑，产品目前可以说完全由lua脚本驱动。Lua脚本通过交互栈（lua状态机）驱动主程序。这个交互栈由luamanager来创建和管理。Luamanager的实例有主程序ui来创建，同时主程序将自己的句柄（指针）传递给luamanager。这样在luamanager以及他的子模块中就可用主程序的句柄来利用Qt反射机制或信号和槽机制实现与主程序的交互。

扩展交互功能，基本操作同扩展lua基本功能的两种操作

2.1 在已存在的类中添加交互功能

需在函数定义中使用已存的ui句柄来与主程序交互

（eg： int LuaHistory::length(lua_State *lua)

{

int number = 0; QMetaObject::invokeMethod(m_pDetailV,"HistoryLength",Qt::DirectConnection,Q_RETURN_ARG( int,number));

lua_pushnumber(lua, number);

return 1;

}）

m_pDetailV 为类中已存的句柄，如果句柄为NULL则反射机制执行失败。

2.2 在新建的类中添加交互功能

基本操作同lua基本功能在新建类中的添加操作

第一步，添加成员变量static QWidget *m_pDetailV;

添加成员函数 void setContainerWidGet(QWidget *pDetailV);

void 新建类名::setContainerWidGet(QWidget *pDetailV)

{

if (pDetailV != NULL)

m_pDetailV = pDetailV;

}

第二步，在 LuaManager::setContainerWidGet(QWidget *pDetailV)方法中添加相应语句新建类名::Instance()->setContainerWidGet(pDetailV);
