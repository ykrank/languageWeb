1 设计说明

1.1 背景

目前客户端与服务器的交互主要是通过lua实现。在现有情况下不能满足时，就需要扩展lua功能或增加lua接口。

1.2 目的

指导Android客户端开发人员，快速在现有接口的基础上扩展lua功能或新增lua接口。

2 Lua扩展

Lua扩展主要分为静态扩展、动态扩展、扩展Lua接口、控件扩展执行Lua几种方式。

2.1 静态扩展

通过方法addJavaInterface()扩展，也称静态扩展。

2.1.1 Step1:创建Java类，实现其方法

public class LuaSpecial {
    …
    /**
     * 显示ALERT
     * 
     * @param msg
     */
    public void showAlert() {
        mEMPRender.alert(null, "这是在Java代码中弹出的Alert！");
    }
    …
}

2.1.2 Step2:添加其对象到Lua对象中

public class MainActivity extends EMPActivity {
    …
    @Override
    public void onAddLua(EMPLua empLua) {
        empLua.addJavaInterface("luaspecial", new LuaSpecial());
    }
    …
}

2.1.3 Step3:应用

<content>
<head>
    <script>
        luaspecial: showAlert();
    </script>
</head>
</content>

2.2 动态扩展

通过上节静态扩展方式加入的Java对象，再通过Lua脚本访问其方法返回Java对象方式称为动态扩展。

2.2.1 Step1:获取需要访问的Java方法和所在的类

lua中将要访问的方法前需要加emp后缀，在应用示例中有详细说明。

public class Test {

    public double empadder(int a, float b) {
        BigDecimal ba = new BigDecimal(Integer.toString(a));
        BigDecimal bb = new BigDecimal(Float.toString(b));
        return ba.add(bb).doubleValue();
    }

    public double sub(int a, float b) {
        BigDecimal ba = new BigDecimal(Integer.toString(a));
        BigDecimal bb = new BigDecimal(Float.toString(b));
        return ba.subtract(bb).doubleValue();
    }
}

2.2.2 Step2:创建Java类，添加相关方法

如在上例中LuaSpecial类中添加createTest()方法：

/*
 * 创建Test对象
 * 
 * @param msg
 * @return
 */
public Object createTest() {
    return new Test();
}

2.2.3 Step3:同上添加到Lua对象中

代码参见静态扩展 step2。

2.2.4 Step4:应用和注意事项

通常此种注册Java对象方式极为不安全，因为返回的Java对象所有公共方法都暴露给Lua。

为解决上述问题，现定义关键字emp。如果在Lua中需要访问以动态扩展方式添加的Java类中的方法，那么该方法名必须已emp开头，否则将不对该方法做处理。

如上述Test类，在Test类中存在empadder与sub两方法，在Lua中empadder方法将可以访问，而sub方法则不可以。

local o = luaspecial:createTest();
if o == nil then 
    window:alert("返回Java对象为null！", "确定");
else
    local c = o:adder(a, b); 
    local d = o:sub(a, b);   
    window:alert("算数运算：".. a .."+".. b .." = ".. c , "确定");
end;

上例中由于adder方法，Test类中方法名是empadder方式，所以正确返回a+b的值c；sub方法由于Test类中方法名未加emp所以sub方法将找不到，故会因d不存在而报Lua错误。

2.3 扩展Lua接口

对已有lua接口新增方法。例如：window接口下新增方法window:test();

前提：
windows下 Android NDK 环境配置。（说明：NDK 是一套交叉编译工具，它可以把用C或C++书写的代码，编译为.so格式的文件，以便在Android 程序中用java语言（JNI）调用这些代码）
参考：windows下androidNDK环境配置

2.3.1 Step1:新建LuaWindow类，并添加方法test

LuaWindow类代码如下：

package com.rytong.emp.test.lua;

import com.rytong.emp.lua.EMPLua;
import com.rytong.emp.lua.EMPLuaFactory;

public class LuaWindow {

    public static void test(int luaIndex){
        // 弹出提示框
        EMPLua empLua = EMPLuaFactory.getEMPLua(luaIndex);
        empLua.getEMPRender().alert(null, "window test ...");
    }
}

2.3.2 Step2:jni\lua目录下添加对应window.c及window.h文件

window.h文件内容：

#ifndef _app_window_h
#define _app_window_h

#include "../emp-lua/emplua.h"

void app_window_register(lua_State * lua);

#endif

window.c文件内容：

#include "window.h"

jclass getClassLuaWindow(JNIEnv * env){
    // LuaWindow 路径 (包名+类名)
    return (*env)->FindClass(env, "com/rytong/emp/test/lua/LuaWindow");
}

int window_test(lua_State * lua){
    JNIEnv * env = getEnvFromLua(lua);
    log(env, "window test");

    int luaindex = getEMPLua(lua);
    jclass cls_window = getClassLuaWindow(env);
    jmethodID mhd_test = (*env)->GetStaticMethodID(env, cls_window, "test", "(I)V");
    (*env)->CallStaticVoidMethod(env, cls_window, mhd_test, luaindex);

    (*env)->DeleteLocalRef(env, cls_window);
    return 0;
}

const struct luaL_reg app_window_libs [] = {
    {"test", window_test},// "test":新增方法名
    {NULL, NULL}
};

void app_window_register(lua_State *lua) {
    // "window" :lua 接口名
    luaL_register(lua, "window", app_window_libs);
}

2.3.3 Step3:在jni\applua.h 文件中引用window.h文件

applua.h文件内容：

#ifndef _applua_h
#define _applua_h

#include "emp-lua/emplua.h"
#include "lua/window.h"

#endif

2.3.4 Step4:在jni\Android.mk文件中添加window.c文件引用

Android.mk文件内容：

LOCAL_PATH := $(call my-dir)

# 当前模块名称

2.3.5 Step5:在jni\applua.c文件，调用方法app_ window_register(lua);

applua.c 文件内容：

    log(env, "addLua!");
    lua_State * lua = getLuaStateFromJNIEnv(env, clua);

    app_window_register(lua);

2.3.6 Step6:重新编译生成libapplua.so文件，并添加至libs\armeabi目录下

编译方法：

1）运行cygwin，进入到当前项目jni文件夹所在目录。
例如：D:\workspace_other\TestAPP

2）输入$NDK/ndk-build。执行成功后，会自动生成一个libs->armeabi目录，把编译生成的.so文件放在里面。($NDK是调用之前配置好的环境变量，ndk-build是调用ndk的编译程序)

注意：若libs->armeabi文件夹已存在，编译时会先清空armeabi中的文件，再生成.so库。记得先备份armeabi下其他文件。

2.4 控件扩展执行Lua

通过扩展Lua使Lua脚本可以操作控件方法，如：button:refresh()。

2.4.1 Step1：在Java代码中实现需要执行的的控件方法

新建一个控件用于扩展，也可以扩展原有控件方法。这里新建一个InputSpecial控件。

package com.rytong.emp.speciallua.view;

public class InputSpecial extends InputButton {

    private Layout mLayout;

    public InputSpecial(Context context) {
        super(context);
    }

    @Override
    public Layout onBuildLayout() {
        mLayout = super.onBuildLayout();
        return mLayout;
    }

    public void alert() {
        mLayout.getEMPRender().alert(null, "这是控件扩展lua执行 alert() 弹出的Alert！");
    }

    public void alert(int a) {
        mLayout.getEMPRender().alert(null, "这是控件扩展lua执行 alert(int) 弹出的Alert！");
    }

    public void showCSS(String name, String value) {
        String msg = "这是控件扩展lua执行 showCSS() 弹出的Alert！样式 " + name + " 的值为：" + value;
        mLayout.getEMPRender().alert(null, msg);
    }
}

2.4.2 Step2:新建LuaMetatableSpecial类，并添加方法

LuaMetatableSpecial类代码如下：

package com.rytong.emp.speciallua.lua;

import android.view.View;

import com.rytong.emp.lua.LuaMetatable;
import com.rytong.emp.speciallua.view.InputSpecial;

public class LuaMetatableSpecial {

    /**
     * metatable:alert()
     * 
     * @param metatable
     */
    public static void alert(LuaMetatable metatable) {
        View view = metatable.getView();
        if (view instanceof InputSpecial) {
            ((InputSpecial) view).alert();
        }
    }

    /**
     * metatable:showCSS()
     * 
     * @param metatable
     * @param name
     * @param value
     */
    public static void showCSS(LuaMetatable metatable, String name, String value) {
        View view = metatable.getView();
        if (view instanceof InputSpecial) {
            ((InputSpecial) view).showCSS(name, value);
        }
    }
}

2.4.3 Step3:jni\lua目录下添加对应metatable.c及metatable.h文件

metatable.h文件内容：

#ifndef _app_metatable_h
#define _app_metatable_h

#include "../emp-lua/emplua.h"
#include "../emp-lua/metatable.h"

void app_metatable_register(lua_State * lua);

#endif

metatable.c文件内容：

#include "metatable.h"

jclass getClassLuaMetatable(JNIEnv * env){
    // LuaMetatable 路径 (包名+类名)
    return (*env)->FindClass(env, "com/rytong/emp/speciallua/lua/LuaMetatableSpecial");
}

int alert(lua_State * lua){
    JNIEnv * env = getEnvFromLua(lua);
    log(env, "luaMetatable alert");

    if (lua_isuserdata(lua, 1)) {
        jobject obj_metatable = *(jobject *) lua_touserdata(lua, 1);

        jclass cls_metatable = getClassLuaMetatable(env);
        jmethodID alert = (*env)->GetStaticMethodID(env, cls_metatable, "alert", "(Lcom/rytong/emp/lua/LuaMetatable;)V");
        (*env)->CallStaticVoidMethod(env, cls_metatable, alert, obj_metatable);

        (*env)->DeleteLocalRef(env, cls_metatable);
    }
    return 0;
}

int showCSS(lua_State * lua) {
    JNIEnv * env = getEnvFromLua(lua);
    log(env, "luaMetatable showCSS");

    const char * name = lua_tostring(lua, -2);
    const char * value = lua_tostring(lua, -1);
    if (name && value && lua_isuserdata(lua, 1)) {
        jobject obj_metatable = *(jobject *) lua_touserdata(lua, 1);
        jstring str_name = (*env)->NewStringUTF(env, name);
        jstring str_value = (*env)->NewStringUTF(env, value);

        jclass cls_metatable = getClassLuaMetatable(env);
        jmethodID alert = (*env)->GetStaticMethodID(env, cls_metatable, "showCSS", "(Lcom/rytong/emp/lua/LuaMetatable;Ljava/lang/String;Ljava/lang/String;)V");
        (*env)->CallStaticVoidMethod(env, cls_metatable, alert, obj_metatable, str_name, str_value);

        (*env)->DeleteLocalRef(env, cls_metatable);
        (*env)->DeleteLocalRef(env, str_value);
        (*env)->DeleteLocalRef(env, str_name);
    }
    return 0;
}

const struct luaL_reg app_metatable_libs [] = {
    {"alert", alert},
    {"showCSS", showCSS}
};

void app_metatable_register(lua_State *lua) {
    // "window" :lua 接口名
    JNIEnv * env = getEnvFromLua(lua);
    int length = sizeof(app_metatable_libs) / sizeof(app_metatable_libs[0]);
    setMetatableSpecial(app_metatable_libs, length);
}
