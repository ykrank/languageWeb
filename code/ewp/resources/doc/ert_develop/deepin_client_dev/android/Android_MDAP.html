Android统计分析使用指南

1 概述

借助于此功能扩展，我们可以统计和分析移动应用的流量来源、内容使用、用户属性和行为数据等，以便进一步利用该数据进行产品、运营、推广策略的决策。

2 环境支持

此功能扩展要求产品EMP为5.2版本及以上，统计后台MDAP的SDK为2.0版本及以上。

3 使用

3.1 代码获取

要使用统计分析功能，有两种方式可以获取到统计分析的代码。

  1. 在获得授权后，前往https://dev.rytong.me:9998/hg/proj/mdap/sdk-android获取Android平台统计分析客户端代码。其工程名为RYTTrack。
  2. 直接找MDAP开发人员要已经打好的jar包，包括ryttrack.jar和libryttrack.so两个文件

3.2 代码导入

针对代码获取方式的不同，也有两种导入统计分析代码的方式。

3.3 修改主工程代码

统计分析代码成功导入主工程后，需要开启统计分析功能，只需要修改主工程的MainActivity类即可。

  1. 修改onAddLua方法，加载libryttrack.so库文件。

     @Override
 public void onAddLua(EMPLua empLua) {
     // 加载.so文件
     try {
         ...
         System.loadLibrary(TrackLua.TRACKLUA_LIB);
     } catch (UnsatisfiedLinkError e) {
         Utils.printException(e);
     }
     // 注册lua 接口\方法
     ...
     TrackLua.addLua(empLua.getCLua());
     ...
 }

  2. 修改onStart、onPause、onRestart、onDestory方法，用于随着Activity的生命周期，开启、暂停、执行或关闭统计分析过程。

     @Override
 protected void onStart() {
     super.onStart();
     if (EMPConfig.newInstance().isTrack()) {
         // 开启统计
         // 统计功能--是否输出Error信息
         Control.OUTPUT_ERROR = EMPConfig.mIsPrintLog;
         // 统计功能--是否输出debug信息
         Control.OUTPUT_DEBUG = EMPConfig.mIsPrintLog;
         // 统计功能--连接服务地址
         Control.TRACK_ADDRESS = "http://192.168.xx.xxx:xxxx/stats/collect";
         // 设置统计用的客户端应用下载渠道
         Control.DOWNLOAD_CHANNEL = "emp";
         TrackAgent.onStartSession(this, null);
     }
 }

 @Override
 protected void onPause() {
     super.onPause();
     if (EMPConfig.newInstance().isTrack()) {
         TrackAgent.onPauseSession();
     }
 }

 @Override
 protected void onRestart() {
     super.onRestart();
     if (EMPConfig.newInstance().isTrack()) {
         TrackAgent.onResumeSession();
     }
 }

 @Override
 public void onDestroy() {
     // 结束统计。
     if (EMPConfig.newInstance().isTrack()) {
         TrackAgent.onEndSession();
     }
     super.onDestroy();
 }

  3. 在onAdjustEMPConfig方法中添加统计开关配置。

     @Override
 public void onAdjustEMPConfig(EMPConfig empConfig) {
     super.onAdjustEMPConfig(empConfig);
     ...
     empConfig.setTrack(true);
     empConfig.setEMPTrackTool(new TrackTool()); // 设置一个统计工具类的实例。
 }

    只有empConfig.setTrack为true时，统计分析功能才会打开

3.4 统计的一些配置

在RYTTrack工程中，文件com.rytong.track.control.Control为统计分析的配置文件，主要配置项如下，可以根据需求灵活配置：

/**
 * <p>进行统计分析时候的控制类。</p>
 */
public class Control {

    /** ApiKey */
    public static String APIKEY = "aaaaaaaa";
    /** 统计接口 */
    public static String TRACK_ADDRESS = "http://192.168.xx.xx:xxxx/stats/collect";
    /** Session超时时间，单位为分钟 */
    public static int SESSION_TIMEOUT = 5;
    /** 发送报文的时间间隔，单位为秒 */
    public static int SEND_TIME_INTERVAL = 300;
    /** 是否输出Error信息 */
    public static boolean OUTPUT_ERROR = true;
    /** 是否输出debug信息 */
    public static boolean OUTPUT_DEBUG = true;
    /** 计算地理位置时候的监测时间间隔。单位：毫秒 */
    public static int LOCATION_TIME = 5 * 1000;
    /** 计算地理位置时候的监测距离，当距离改变量大于这个值的时候进行一次监测。单位：米 */
    public static int LOCATION_DISTANCE = 500;
    /** 应用下载渠道  */
    public static String DOWNLOAD_CHANNEL = "";
}
