Windows Phone客户端控件扩展说明

产品中已经封装了在开发中可能需要使用的控件，包括基础控件和一些高级组合控件，但已有的控件也可能不能满足项目开发的需求。本文档用以指导根据现有的代码结构如何新增自定义的控件或进行控件扩展。

控件继承结构说明

产品所有的控件都是一个自定义封装类，而不是Windows Phone中Silverlight类型的控件，所有控件直接或间接继承自RYTControl这个控件基类，RYTControl类型有一个公开属性View_属性，这个属性就是指RYTControl生成之后的FrameworkElement，也就是Silverlight控件，也就是说RYTControl类型实际上是页面上显示的UI元素的逻辑封装。以下是RYTControl控件基类主要成员说明：

1 字段和属性：

名称 说明
CurrentCSSStyle_ 控件的样式，报文解析后的会给该对象赋值
Name_ 控件的名称
Parent_ 控件的直接父亲（RYTControl类型）
Value_ 一般用来表示控件的展示或者表单提交值
XElement 控件报文的Xml节点对象
View_ 控件的UI视图元素
Text_ 控件的文本值

2 方法或事件

名称 说明
ApplyStyle 应用控件的样式
BuildChildControlAction 对于容器控件进行排版布局
BuildChildControls 对于容器控件进行视图构建与布局
Focus 控件是否已获得焦点
GetAttributeValueWithAttName 根据属性名称获取该属性的值
GetDefaultOrInputOrSelectedValue 获取控件的值
InitView 初始化控件视图
LayoutSubviews 布局控件和构建子控件

如何新增用户控件或扩展已有控件

1 新建控件类型

  1. 在扩展控件库中，创建一个新的类，继承基类RYTControl，注意命名规范（形如RYT\****Control）。
  2. 根据控件的UI特点和功能选择性重写基类的虚方法，如void InitView()， void BuildChildControls(), void ApplyStyle()。
  3. 在InitView方法中，可将扩展控件所构造出来的UI对象（silverli控件）的引用赋值给View_属性。
  4. 在ApplyStyle方法中应用控件的CSS模版指定的样式. 根据Xml结构由外向内生成控件，执行InitView()方法和ApplyStyle()方法进行控件初始化。
  5. 初始化全部控件后用BuildChildControls()由内向外布局子控件，最后放进Page中。

2 更改class.xml文件完成控件和标签的映射

在MainProject工程下的resource目录下的class.xml是用来配置控件和标签的映射类的，将新编写的扩展控件信息加入到class.xml中即完成关联。

例如：

<object class="RYTLinePlotControl" init="initWithParam" tag="ul" type="ballPlot">
<param key="Frame">0.0,0.0,10.0,10.0</param>
<param key="Hidden">NO</param>
<param key="Enable">YES</param>
</object>

当接受到后台编写报文中含有<ul type="ballPlot" name="plot" class=".plot" />时，即会生成控件库中的RYTLinePlotControl控件。
