Windows Phone数据采集（MDAP）概述说明

统计客户端用户使用行为，如用户点击操作，页面浏览次数等。

统计类库说明

1 客户端统计数据库表结构：

目前客户端主要由三张数据表持久化用户数据：

InfoTable

用户信息表：主要保存如用户设备号，用户id等信息

SessionTable

会话状态表：保存当前一个会话的信息，如SessionId，PageViews，以及会话暂停，持续时间等信息。另外关联EventTable，ErrorTable的主外键关系字段。

EventTable

事件表：保存事件的名称和发生时间信息

ErrorTable

错误信息表：

2 统计数据生成：

统计数据由查询表数据如InfoTable，SessionTable等遍历表数据生成Json格式字符串发送给MDAP服务器。

客户端调用MDAP类库的TrackSeeker.Seek()方法生成统计数据。经加密位移运算后发送给MDAP服务器。

Windows Phone数据采集（MDAP）类库使用说明

1 类库引用添加

在主工程项目上引用RYTTrackLib类库，此类库包含所有关于信息采集所需的逻辑，如数据库及表的创建，表数据的查询遍历，数据的加密处理等。

备注：也可以不添加程序集引用，但必须将RYTong.RYTTrackLib.dll程序集包含于主工程bin目录中，程序集是以插件的形式在运行时动态创建和解析的。

2 关于数据采集的入口方法

RYTTrack.StartSession（string apiKey）是程序的入口方法。DMAP对此采用单例模式封装了RYTTrack实例，使用时请直接使用RYTTrackLib.RYTTrack.Instanse.StartSession("26981JUU")调用。

3 关于在主工程项目中的使用

在Hostpage.xaml.cs后台类代码中，在其构造函数中调用

RYTMainHelper.InitTrack(List appNavigatedToActionList, List appNavigatedFromActionList);，并确保打开统计行为开关 bTrackEnable=true

说明：appNavigatedToActionList是页面切入时的调用方法列表，appNavigatedFromActionList是页面切出时的方法列表。

调用方法已在RYTMainHelper类中进行了封装，请直接使用！

关于apikey的说明：

apikey是统计接口入口参数，是服务器分配给客户端的标识。关于其详细使用请参考统计分析相关文档。
使用时要更改apikey参数，请修改主工程静态配置类字段Track_apiKey，并确保在调用RYTMainHelper.InitTrack（）方法前正确更改了此值。

统计接口数据发送服务器地址：

地址由静态配置类字段 TrackServerUrl指定，ip和端口号请根据实际使用情况指定
