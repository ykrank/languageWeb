如何扩展Lua接口

1 5.1及以前版本：

步骤一：

在MainProject或新建工程中，编写相应Lua接口类，如LuaFile.cs,并为其编写public static int Luaopen_filelibs(Lua.lua_State lua)的静态注册方法。

以下以LuaFile接口为例：

public class LuaFile

{

private LuaFile() { }

static int write(Lua.lua\_State lua) //实际调用方法

{ XXXXXXX ;return 0; }

static int read(Lua.lua\_State lua)

{XXXXXXXXX ;Lua.lua\_pushnil(lua);return 1;}

   private static Lua.luaL\_Reg[] file\_libs = {

    new Lua.luaL\_Reg("write",write), //注册关联方法

    new Lua.luaL\_Reg("read",read),

    new Lua.luaL\_Reg(null, null) };

   public static int luaopen\_filelibs(Lua.lua\_State lua)

   {

     Lua.luaL\_register(lua, "file", file\_libs); //方法库名

     return 1;

   }

 }

步骤二：

在程序入口（HostPage构造函数）中，

执行以下代码 ：

var luaInstance = RYTong.LuaScript.LuaManager.Instance；

LuaFile.Luaopen\_filelibs(luaInstance.Lua\_State);

即可完成对向Lua状态机对LuaFile接口方法库的注册。

2 5.2及以后版本：

大体流程： C++/CX（运行时）通过调用luaL_register(lua, “window”, libs)的方式将 C++/CX 中的代理注册到Lua环境中，再通过public static void SetCplusDelegates() 将C++中的代理绑定到具体的C#实现代码上，这样运行时首先去注册绑定好的代理，lua执行方法时则会自动去调用我们的实现代码。下面看下具体的实现步骤。

Step1：添加运行时扩展

在LuaCplusExtend 工程中，编写用于绑定C# 方法的运行时代理，如LuaWindowEDelegates.h，然后再添加用于向lua中注册方法的扩展类，如：LuaWindowE.cpp，编写 ClassName::registerLib(lua_State *lua) 方法，调用 luaL_register(lua, 扩展名, 扩展方法的映射集合) 将运行时中的代理注册到lua环境中。 如：

const struct luaL_Reg libs [] =
{
    /*  “show” 为调用方法名， LuaWindowE::show为方法名*/
    { "show", LuaWindowE::show },
    { "alert", LuaWindowE::alert },
    { "close", LuaWindowE::close },
    { "open", LuaWindowE::open },
    { "showContent", LuaWindowE::showContent },
    { "showControl", LuaWindowE::showControl },
    { "hide", LuaWindowE::hide },
    { "setPhysicalkeyListener", LuaWindowE::setPhysicalkeyListener },
    { "closeKeyboard", LuaWindowE::closeKeyboard },
    {NULL,NULL}
};

int LuaWindowE::registerLib(lua_State *lua)
{
    luaL_register(lua, "windowE", libs);
    return 1;
}

// 这样我们在 lua脚本使用时应为 接口名：注册的方法名（） 如： windowE：show();

在头文件pch.h 中添加上面新增的 代理头文件和扩展类头文件。 最后在 Lua.cpp 的 openLua(void)方法中调用扩展类的 registerLib(lua_State *lua) 方法；

Step2：C++/CX代理与C#方法绑定

在LuaScriptExtend工程中，编写相应Lua扩展方法实现类，如LuaWindowE.cs,并为其编写 public static void SetCplusDelegates()的静态方法。将step1中的代理与相应的实现方法绑定。

以WindowE接口为例：

    public static void SetCplus_Delegates()
    {
        LuaWindowEDelegates.showD = new ShowDel(() => { MessageBox.Show(" show Test "); return 1; });
        LuaWindowEDelegates.alertD = new AlertDel(alert);
        LuaWindowEDelegates.close = new LuaCplusExtend.closeDel(close);
        LuaWindowEDelegates.open = new LuaCplusExtend.openDel(open);
        LuaWindowEDelegates.showControl = new LuaCplusExtend.showControlDel
        ...
    }

最后在 DelegatesManager 类的SetAllDelegates() 方法中添加新增的绑定。 即完成对Lua状态机的方法库注册。

    public static void SetAllDelegates()
    {
        LuaE.openLua();
        LuaFileE.SetCplusDelegates();
        LuaWindowE.SetCplus_Delegates();
    }

Step3:C#方法中参数与返回值的使用

一. 参数校验

由于方法的参数是事先定义好的，我们可以通过调用下面的方法按照顺序传入参数类型进行校验，如果失败返回 false； LuaCommon.CheckAndShowArgsError(LConst.String，………)

二．参数获取

参数的获取是通过lua的堆栈元素去取得，从第二位开始为参数数据，根据参数类型 通过调用以下方法：

  • Lua.Lua_tostring(2)
  • Lua.Lua_touserdata(3)
  • Lua.Lua_tointeger(4)
  • …

来分别获取

  • 第一个string类型的参数，
  • 第二个 object类型的参数，
  • 第三个整型参数。 以此类推。
  • …

获取参数之前也可以通过 Lua.Luais*() 等方法来判断数据类型是否正确 如：

  • Lua.Lua_isstring(number) // 是否为字符串类型
  • Lua.Lua_isfunction(-1) // 是否为回掉方法。

回掉的使用可以参考 LuaWindowE.alert();

三．返回值

真正的数据返回依然通过lua堆栈实现：

  • Lua.Lua_pushnil(); 返回 nil 空值
  • Lua.Lua_pushstring(“”) 返回一个字符串
  • Lua.Lua_pushlightuserdata(object) 返回一个object类型数据
  • ……

具体的实现可参考事例：LuaFileE.cs 和 LuaWindowE.cs (代码通过反射调用真正的需要操作的代码，自己可以按情况实现)；

上面是反回给lua的，而代码中return的都是整数类型，表示返回给Lua堆栈数据的个数

  • Return 0 即为void 没有需要返回的数据。
  • Return 1 即返回了一个参数。
  • ….以此类推
