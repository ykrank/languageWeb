ERT开发

iOS开发模式使用

1 简介

调试模式是 iOS EMP 提供的一个用于开发工作的功能，使用该功能可主动下载本地没有的资源文件。默认情况下，调试模式是关闭的，所以需要在配置文件中手动打开它。

2 Config.h 配置

Config.h 文件关于调试模式的宏定义与简介如下：

/** 开发调试
  * APP_TEST及其包含的LOAD_前缀的宏定义,都是在开发模式下使用的宏定义.
  * 请在生产环境中关闭此功能.
  * APP_TEST功能的作用是在开发模式下,主动下载本地没有的资源文件.
  * @see LOAD_LUA_FILE
  * @see LOAD_IMAGE_FILE
  * @see LOAD_CSS_FILE
  * @see LOAD_XHTML_FILE
 */
#ifndef APP_TEST
//    #define APP_TEST
#endif
#ifdef APP_TEST
    #define LOAD_LUA_FILE // 控制lua文件的下载
    #define LOAD_IMAGE_FILE // 控制IMAGE文件的下载
    #define LOAD_CSS_FILE // 控制CSS文件的下载
    #define LOAD_XHTML_FILE // 控制XHTML文件的下载
#endif

3 RYTConfig.h 配置

RYTConfig.h 文件需添加以下方法声明：

// 是否开启开发调试模式
- (BOOL)appTest;
// 是否加载Lua文件
- (BOOL)loadLuaFile;
// 是否加载CSS文件
- (BOOL)loadCssFile;
// 是否加载Image文件
- (BOOL)loadImageFile;
// 是否加载Xhtml文件
- (BOOL)loadXhtmlFile;

4 RYTConfig.m 配置

RYTConfig.m 文件需要添加以下方法实现：

- (BOOL)appTest {
#ifdef APP_TEST
    return YES;
#else
    return NO;
#endif
}
- (BOOL)loadLuaFile {
#ifdef LOAD_LUA_FILE
    return YES;
#else 
    return NO;
#endif
}
- (BOOL)loadImageFile {
#ifdef LOAD_IMAGE_FILE
    return YES;
#else
    return NO;
#endif
}
- (BOOL)loadCssFile {
#ifdef LOAD_CSS_FILE
    return YES;
#else
    return NO;
#endif
}
- (BOOL)loadXhtmlFile {
#ifdef LOAD_XHTML_FILE
    return YES;
#else
    return NO;
#endif
}

5 调试模式下数据请求接口

在上述文档中提到，APP_TEST宏定义的开启可以控制客户端主动下载本地没有的资源文件。客户端请求的EWP接口如下：

接口方法http://EWP/test_s/get_page（EWP为EWP服务器地址）
HTTP请求方法POST/GET
描述在开发模式下，如果客户端在本地没有找到需要的相关资源则会请求该接口获取资源。
接口需求方客户端
接口提供者EWP服务器
参数（加密信道中传输）
name如果资源为普通资源，则资源路径形如：name=channels/base.png;
如果资源为插件资源，则资源路径形如：name=channels/login/images/base.png;
其中，在资源路径前拼接的channels/为与服务器商定的必须字段。
platform客户端平台：iphone
resolution屏幕分辨率，格式为：widthheight，比如：320480

6 其他

针对调试模式的使用，做以下几点说明：

  1. 资源请求为同步请求，如果请求成功，则返回成功的数据。如果请求失败，则在控制台日志中，会看到类似于加载远程资源发生错误的日志输出，并会打印出服务器返回的错误信息。开发人员可以根据错误日志排查错误。
  2. 调试模式下请求的资源，并没有同本地离线资源一样做SHA1校检，因此是不保证安全的。请在生产环境下不要使用该功能。
