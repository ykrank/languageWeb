1 config配置简介

1.1 常用调试开关

1.1.1 NATIVE_DEVELOP本地调试开关

打开此宏定义,可以加载显示测试文件的报文.用法查看

#define NATIVE_DEVELOP

1.1.2 NATIVE_DEVELOP_FILE 本地调试文件名称

默认后缀为.xml，使用前提需要先定义宏NATIVE_DEVELOP.用法查看

#define NATIVE_DEVELOP_FILE @"analyseTest"

1.1.3 LOG 日志输出控制

控制ZNLog输出的日志.打开LOG,会将日志输出到控制台.关闭LOG,将不会输出日志.用法查看

#define LOG

1.1.4 CONSOLE 真机设备上显示控制台信息

当需要给脚本开发人员展示控制台信息时,可以通过此功能在手机页面上打印控制台信息.用法查看

两个条件:1. 必须在真机上.2. 定义DEBUG宏.3. 打开宏定义LOG.

DEBUG宏定义方式:可以在TARGETS APP的Build Settings中,查找Preprocessor Macros字段,在Debug中加入值DEBUG.

#define CONSOLE

1.1.5 TEMPLATEMODE 用户访问模式

用法查看

#define TEMPLATEMODE

1.1.6 CHECK_LUA_ERROR CHECK_CSS_ERROR CHECK_XML_ERROR 调试错误检测

此功能通常用户开发人员在调试时的错误检测.建议在生产环境中关闭此功能.用法查看

#define CHECK_LUA_ERROR  //用来检测并提示lua的错误信息.
#define CHECK_CSS_ERROR  //用来检测并提示样式的错误信息.
#define CHECK_XML_ERROR  //用来检测并提示XML的错误信息.

1.1.7 APP_TEST 开发调试

在生产环境中关闭此功能.APP_TEST功能的作用是在开发模式下,主动下载本地没有的资源文件.考虑到效率的问题,下载是不保证安全性的.用法查看

   #define APP_TEST

1.1.8 LOAD_LUA_FILE

控制lua文件的下载.使用需提前定义#define APP_TEST.用法查看

#define LOAD_LUA_FILE

1.1.9 LOAD_IMAGE_FILE

控制图片文件的下载.使用需提前定义#define APP_TEST.用法查看

#define LOAD_IMAGE_FILE

1.1.10 LOAD_CSS_FILE

控制CSS文件的下载,使用需提前定义#define APP_TEST.用法查看

#define LOAD_CSS_FILE

1.1.11 LOAD_XHTML_FILE

用法查看

#define LOAD_XHTML_FILE

1.1.12 START_USER_BEHAVIOUR_ANALYSE 统计功能开关

通过START_USER_BEHAVIOUR_ANALYSE控制统计功能的开启与关闭如果需要开启统计功能,则打开此宏定义.反之,关闭.用法查看

#define START_USER_BEHAVIOUR_ANALYSE

1.1.13 Editor 动态交互式调试

用法查看

#define EMP_EDITOR

1.2 企业应用商店相关设置

1.2.1 ENTERPRISE_STOR_SERVER_URL

当installApp中提供的下载路径不是以http或https开头的,程序会默认使用ENTERPRISE_STOR_SERVER_URL中定义的服务器地址.作为应用下载地址.用法查看

#define ENTERPRISE_STOR_SERVER_URL @"https://192.168.64.127:443"

1.2.2 CALLBACK_TIME_INTERVAL

下载app时回调lua监听函数的时间间隔,如果回调间隔时间为0，则默认为30秒；如果间隔时间大于最长的安装时间MAX_INSTALL_TIME，则取最长时间作为回调时间.用法查看

#define CALLBACK_TIME_INTERVAL 5

1.2.3 MAX_INSTALL_TIME 最长的安装时间

单位秒,如果超过这个时间没有安装成功,则认为应用安装失败.用法查看

#define MAX_INSTALL_TIME 1800

2 RYTConfig配置简介

2.0.1 - (BOOL)needLowerCase;

返回宏NEED_LOWER_CASE

密码键盘是否需要大小写转换,根据宏NEED_LOWER_CASE是否定义来返回YES或者NO.

2.0.2 - (NSString *)serverURL;

返回宏SERVER_URL

服务器地址,如果定义了宏TEMPLATEMODE的话,将会进入自定义的用户访问页面,否则直接进入宏SERVER_URL定义的服务器地址.除此之外,在加载远程资源,下载网络图片等网络需求的情况,都会调用此方法,将SERVER_URL定义的服务器地址进行拼接.

2.0.3 - (NSString *)appName;

返回宏APP_NAME

请求的URL中参数app的值,直接返回宏APP_NAME的值.

2.0.4 - (BOOL)encryption;

返回宏ENCRYPTION

是否建立加密信道（当前判断仅在Debug模式下有效，否则一直返回YES）(目前没有作用)

2.0.5 + (BOOL)isBasedEncryptChannel;

返回宏BASED_ENCRYPT_CHANNEL,SEPARATE_ENCRYPT_CHANNEL

客户端自校验,是否基于信道来处理校验,如果定义宏BASED_ENCRYPT_CHANNEL返回YES,如果定义了宏SEPARATE_ENCRYPT_CHANNEL返回NO,都没有定义返回YES.代码会在TLS里将返回值作为判断条件之一,是否将签名数据拼入请求里.

2.0.6 - (BOOL)nativeDevelop;

返回宏NATIVE_DEVELOP

是否调试本地页面,如果定义了宏NATIVE_DEVELOP,则返回YES,否则返回NO.
注:如果开启本地调试,将会加载本地XML报文.同时无法进行网络图片的下载,form表单提交,注册远程提示,开启离线开关等一系列操作.

2.0.7 - (NSString *)nativeDevelopFile;

返回宏NATIVE_DEVELOP_FILE

本地调试的文件.如果定义了宏NATIVE_DEVELOP及NATIVE_DEVELOP_FILE,那么就会返回宏NATIVE_DEVELOP_FILE的值,否则返回nil.
注:此方法返回的本地XML文件名称,会在根视图初始化时读取对应的本地XML文件,用以显示.

2.0.8 - (BOOL)console;

返回宏CONSOLE

真机是否显示控制台信息.如果定义了宏CONSOLE,则返回YES,否侧返回NO;
注:在真机且debug环境下,会通过对本方法返回值的判断,来决定是否创建一个view来显示控制台信息.

2.0.9 -(BOOL)log;

返回宏LOG

是否输出日志.如果定义了宏LOG返回YES,否则返回NO.
注:此方法会控制所有ZNLog的打印,调试代码时,打开,可在控制台看到打印信息.

2.0.10 - (NSString *)Ad;

返回宏AD

客户端渠道参数.返回宏AD的值.会在TLS里拿来做clientinfo参数拼接.

2.0.11 - (NSString *)ClientID;

返回宏ClientId

客户端标识,返回宏ClientId的值.会在TLS中拿来做clientinfo参数及证书参数拼接.

2.0.12 - (NSString *)OS;

返回宏PLOTFORM

平台标识,返回宏PLOTFORM的值.一般拿来做请求网址的拼接.

2.0.13 - (BOOL)iPhone;

是否为iPhone设备（包括iPod Touch).
