iOS 配置使用说明

1 简介

EMPConfig为工程配置类,EMP提供配置API来对功能进行开启,关闭,设置等操作.
为了兼容RYTConfig的配置方案.本类在初始化时,会默认加载RYTConfig文件中的配置项.仅针对以前版本中需要在RYTConfig类中实现的配置.
如果RYTConfig中没有相关配置,则会只用默认配置.
同时EMP提供统一设置EMP配置的函数.[UIContentViewController EMPSettings];

@see - (void)EMPSettings

2.2 screenSize

@property(nonatomic,assign,readonly) CGSize screenSize;

返回当前屏幕的宽与高

2.3 applicationFrame

@property(nonatomic,assign,readonly) CGRect applicationFrame;

/**
 *   应用程序界面的frame,产品中用它来处理视图的坐标及大小.
 *   UIContentViewController的view有系统决定.不同的系统会有差异.
 *   UIContentViewController中的contentView_的frame由applicationFrame指定.
 *   contentView_作为UIContentViewController的view的子视图.
 *   页面中的body控件,默认大小与contentView_相同.
 *   showControl和showContent页面的frame也来自与applicationFrame.
 *   statusBarHeight 与 applicationFrame具有关联性，其size.height之和等同于screenSize.height
 *   注:在ios 7.0 版本前后,状态栏对contentView_的位置是有影响的.
 */

2.4 fitSize

@property(nonatomic,assign) CGFloat fitSize;

/*针对不同机型,取得适配比例系数
 *处理方式为 屏幕宽度/固定参数 获得一个适配比例系数
 *屏幕宽度参数取值:- (CGSize)screenSize;方法中返回size的宽.
 *固定参数取值:iPhone为320,iPad为1024.(即目前iPhone考虑的是竖屏适配,iPad考虑的横屏适配)
 *代码中通过调用此方法,获得比例系数,从而对控件的大小,位置,字体等等进行适配.
 */

2.5 statusBarHeight

@property(nonatomic,assign) CGFloat statusBarHeight;

/**
 *  获取状态栏高度.这个高度与状态栏的隐藏与显示没有关系.
 *  仅仅用来决定应用程序内容的起始位置.
 *  通过返回的statusBarHeight值得不同,我们可以控制页面内容显示效果,当状态栏高度返回0时,
 *  效果如ios7,当状态栏高度返回20时,效果如ios6.
 *  默认为20.
 *  @return 状态栏应该占用的高度
 */

2.6 supporStatusBarInXML

+ (BOOL) supporStatusBarInXML;

/**
 *  返回是否支持 ios7 状态栏效果. 如果通过模板实现ios 状态栏的效果,这里需要返回 true.
 *  根据 emp 对 ios7状态栏的支持方式,当设置状态栏高度为0时,说明状态栏不占位.
 *  这里没有处理通过调整 UIContentView 的位置的情况.
 *  因此这里的算法并不能覆盖所有的情况. 如果项目中自定义了 UIContentView 的位置,需要重写本接口已返回正确的状态.
 */

注：添加以上代码开启异常捕获功能。开启捕获异常功能只是在沙盒路径Documents下新建了一个crash.log文件，用来记录崩溃及异常。项目当开启捕获异常的功能时，自行维护此文件。
