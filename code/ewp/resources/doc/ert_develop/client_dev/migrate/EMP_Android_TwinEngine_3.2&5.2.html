EMP3.2和EMP5.2双引擎兼容改造方案

1 概述

本文档主要描述在EMP3.2的引擎基础上，兼容支持EMP5.2引擎的改造方法。以下改造方法以邮储银行项目为例，其它EMP3.2框架项目可参考修改。

2 以3.2框架首页为入口的改造方法

首先需要在项目工程中导入5.2框架的jar包。

2.1 更改冲突的文件名

由于邮储项目和5.2框架都加了防篡改功能，且用的so库都为libAppVerify.so，因此需要将项目中的使用的so库改名为libAppVerify1.so，同时将com.rytong.tools.crypto.AppVerify中的System.loadLibrary("AppVerify");代码段修改为System.loadLibrary("AppVerify1");。

2.2 注册5.2引擎

在com.rytong.bankps.BaseView类中增加变量：

private EMPRender mEMPRender = null; 
public static AndroidEMPBuilder mAndroidEMPBuilder = null;

在onCreate()方法中增加以下代码：

mAndroidEMPBuilder = new AndroidEMPBuilder(this); 
mEMPRender = new EMPRender(mAndroidEMPBuilder); 
// 修改设置 
final EMPConfig empConfig = mEMPRender.getEMPConfig(); 
onAdjustEMPConfig(empConfig);

覆写方法onAdjustEMPConfig()，该方法用于设置5.2引擎运行时的一些配置，各配置项含义可参考EmpConfig配置。

/** 
 * 设置运行配置 
 * @param empConfig 
 */ 
public void onAdjustEMPConfig(EMPConfig empConfig) { 
    empConfig.setBasicScreen(new Rect(0, 0, 320, 480));
    empConfig.setPrintLog(true);
    empConfig.setAlertPrompt(true);
    empConfig.setUpdateOfflineRes(true);
    empConfig.setDevelopmentMode(true);
    empConfig.setOfflinePromptOn(true);
    empConfig.setTrack(false);
    empConfig.setServerUri("http://192.168.64.128:4002");
    empConfig.setServerUriNoPort("http://192.168.64.128");
    empConfig.setOfflineVersion(1);
}

覆写方法onWindowFocusChanged()，主要作用是记录activity的可用显示范围大小。

/** 
 * 在获取焦点时调整body并记录在Screen类中 
 */

    super.onWindowFocusChanged(hasFocus);
    if (mAndroidEMPBuilder != null)  {
        mAndroidEMPBuilder.onWindowFocusChanged(hasFocus);
    }
}

3.3 网络连接修改

由于新老框架引擎采用一条信道，但网络连接时使用各自的网络模块，因此需要改造3.2框架的网络模块。

  1. 修改com.rytong.bankps.HttpManager中的addSerialNData方法，保证新老引擎网络请求中的防重放参数保持同步。

     import com.rytong.emp.net.CryptoHttpManager;

 private String addSerialNData(String body) {
     String result = null;
     if (body != null) {
         // 新老引擎网络请求防重放参数保持同步
         if (CryptoHttpManager.mSerialN > serialN_) {
             serialN_ = CryptoHttpManager.mSerialN;
         }
         serialN_ += 1;
         CryptoHttpManager.mSerialN = serialN_;
         byte[] byt = new byte[8];
         for (int i = 0; i < 8; i++) {
             byt[i] = (byte) (serialN_ >> (56 - i * 8));
         }
         String serN = Base64.encode(byt);
         result = serN + body;
     }

     return result;
 }

3.4 开启新业务首页

使用5.2框架实现新业务首页时，只需要将NewEngineActivity设为项目的主Activity即可。或者在原有欢迎页面的Acitivity执行完后，通过代码调用。在需要开启新业务主页的地方添加如下代码：

Intent intent = new Intent();
// 定义值传递对象
Bundle bundle = new Bundle();
intent.putExtras(bundle);
intent.setClass(activity, NewEngineActivity.class);
activity.startActivity(intent);
WaitDialog.Task.cancelProgressBar(activity);
activity.finish();

注意：5.2引擎的Activity，整个程序运行过程中只能存在一份，且必须在程序退出时才可销毁。

 }
