前端开发手册

ERT前端框架介绍

  • 源码目录结构
  • 基础UI库
  • ert.lua文件
  • project UI

前端开发框架由基础UI库、项目UI库和ert.lua脚本库三部分组成。

这些库基于特定的EMP产品版本提供，通常情况下向下兼容，即基于EMP5.2开发的公共库可以在EMP5.3及后续版本使用（反之则不成立）。

不具有向下兼容的公共库应该特殊说明

1 源码目录结构

  • ui-lib
    • ert_ui
    • export_snippets
    • ert_lua
    • project_ui
      • ebank_poc
      • hx_ebank
      • ...

2 基础UI库

定义在ert_ui文件夹下的常用HTML标签和CSS样式库。这些库通过emp-debugger插件集成到atom开发环境中，通过代码自动补全的方式使用。

在定义ert_ui中的CSS样式库时，我们需要遵循一定的命名规则来避免冲突的发生。目前的规则如下：

ert_'标签名'_'功能1'_'功能2'_... ert_button_red ert_button_blue

例如，我们使用ert_div_multi_inputs来表示应用于包含多个input的div之上的样式，而这个代码片段通常会通过键入ediv...来呼出ert_ui中所定义的可选div模板。

基础UI库通常由IDE开发人员集成，我们为不同EMP版本的UI库开发不同的ATOM插件，以区分不同EMP版本可以使用的基础UI库。

后续我们会为项目实施开放基础UI的扩展方式，以提升基础UI的支持范围。

3 ert.lua文件

ert.lua文件通过ert_lua文件夹中的源码生成，其目录结构如下：

  • ert_lua
    • src
      • ert.lua
      • ert.channel.lua
      • ert.query.lua
      • ...
    • Makefile
    • ert.lua

通过make命令将src下的源码生成ert.lua文件，通过make doc命令生成最新的文档到doc目录下。开发人员可以通过src下的模块lua代码类似的方式扩展ert.lua脚本。

ert.lua的文档通过 luadoc 工具创建，如需生成，需要安装相关的环境。

生成的Lua API参考手册如下: ERT Lua API

4 project UI

project UI为各项目定义，通过Atom emp-template-management管理的项目公共UI组件。

我们为每一个项目创建一个独立的template package，以保证项目之间的公共UI组件不存在冲突。例如，我们为华夏银行创建的package名称为hx_ebank，为交行poc创建的package名称为ebank_poc。

在创建package是，我们应该在其描述字段中说明该package支持的EMP版本，即基于哪个版本开发。

在编写项目公共UI组件时，我们为每个UI组件定义一个唯一的ID，将$project_name作为UI组件内部元素的命名前缀。例如：

hx_ebank_title hx_ebank_footer ebank_poc_title ebank_poc_footer

同时，在同一package中不同UI组件中，我们应该为组件开发人员定义xhtml文件、css样式和lua函数的命名规范，以防止其冲突。同理，我们使用UI组件的ID作为该组件元素的命名前缀，例如：

hx_ebank_title.xhtml .hx_ebank_title_div {...} function hx_ebank_title_back()
