2 样式文件命名以及存储路径

说明： 所有界面文件都存储在public/www/resource_dev文件夹下，如果为channel相关文件则存储在 resource_dev/common/channels/channelId下, 例如：resource_dev/common/channels/balance_qry则存储为余额查询功能相关文件。 如果为所有channel公用样式文件则存储在resource_dev/common/css下。关于resource_dev具体文件目录结构说明请参照文件夹目录介绍

  • 一个界面文件对应一个自己的样式文件，约定样式文件和界面文件名称一样,例如:MB1001.xhtml和MB1001.css
  • 当前页面特定样式文件放在当前channel下css文件夹中，例如balance_qry/css/MB1001.css
  • 如果当前channel的多个界面有重复样式，则将重复样式提取出来，命名为channelId.css，例如balance_qry.css, 此文件存储路径为当前channel下css文件夹.
  • 如果为当前项目公共样式，命名为项目名_public.css，例如poc_public.css,此文件存放路径为common下的css文件夹中,例如common/css/poc_public.css, 如果公共控件的样式文件中包含了所有需要提取的项目公共样式，那么此文件可以不用写.

3 公共样式命名规范

3.1 统一规则

对于公共样式我们不使用样式是作用于什么内容命名，比如命名为div_content，这样样式只能作用于内容部分div，我们希望样式是不管在什么地方只要 适用就能使用，所以我们使用面向属性命名，假定样式为 {width:320px;height:50px;backgroud-color:#FFFFFF;}则样式命名为.w320_h50_white。碰到样式属性特别多时，我们也不用一一列举把 样式名称弄得特别长，只用列举一些特殊意义属性即可。

3.2 当前channel公共样式

如果为当前channel公共样式，则规定样式前缀为当前channelId_如果channelId特别长，适当加以简化。例如bal_qry_bg_red。

3.3 所有channel公共样式

对于整个项目公共样式，规定样式前缀为项目命_，例如poc_bg_blue.
