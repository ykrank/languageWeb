1 源文件

  • 文件采用UTF-8编码
  • 文件后缀为小写的"lua"

2 命名

  • 类(class)文件名使用骆驼命名法中下划线法。
    例如：在编写数据库相关公共方法时可以命名为db_util_fun。
  • 变量名,函数名使用下划线法
    例如：
    定义一个变量用户名命名为user_name。
    定义一个函数设置用户名命名为set_user_name。
  • 变量名尽量绕开系统保留字
  • 全局变量使用全大写,以_分割单词
    例如：定义一个全局变量用户是否登录命名为USER_IF_LOGIN
  • 命名尽量用有意义的英文
  • 如果单词过长,可使用缩写
  • 英文单词过于复杂时,才能使用拼音

3 排版

  • 使用'\t'(tab)进行代码缩进
  • 比较符(=,==,>,>=,<,<=),连接符(..),运算符(+,-,*,/,%,^)左右需要有空格
  • 逗号前不需要空格,逗号后需要有空格
  • 括号不需要空格
  • 函数之间用空行隔开
  • 代码块含义不同时,用空行隔开
  • 不允许一次使用多个空行
  • if,for,while等语句独立成行
  • 一次只声明一个变量

4 注释

  • 单行注释用"–"
  • 多行注释用"–[[ --]]"
  • 注释区域
    • 版本,版权说明
    • 类说明,文件说明
    • 函数接口说明
    • 变量,全局变量
    • 重要代码块
  • 注释上方需要空行,下方不需要

6 界面脚本规范

6.1 文件名规范

分几种不同的情况：

  1. 一个界面对应一个自己的脚本文件，约定脚本文件名称和界面名称一样为TranCode.lua。比如MB0101.lua,存放路径为channelId/lua/TranCode.lua。
  2. 如果碰到一个channel中的几个界面都用到的方法可以写在一个文件中同时命名为channelId.lua。比如balance_query.lua，存放路径为 channel/lua/channelId.lua。
  3. 如果为整个项目所用到公共方法则命名为项目名.lua，存放路径为common/lua/项目名.lua。

6.2 方法名规范

同上命名中规范。

对于channel中公共方法，约定定义一个以channelId_为前缀的全局table，例如transfer_util_fun，然后将所有可调用方法存入此table中。 例如：

transfer_util_fun = {}; -转账汇款功能公共方法

--[[
@doc：设置用户账号
--]]
function transfer_util_fun:set_user_acc(...)
    ...
end;

对于项目中公共方法，约定定义一个以项目名_为前缀的全局table，例如poc_unit_fun，然后将所有可调用方法存入此table。

例如：

poc_util_fun = {}; -poc公共方法

--[[
@doc：
--]]
function poc_util_fun:format_money(...)
    ...
end;
