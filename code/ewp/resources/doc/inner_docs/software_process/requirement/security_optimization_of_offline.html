2 引言

2.1 目的

本文旨在说明离线协议安全优化的具体需求，使得设计人员和开发人员理解需要实现的具体功能，并为测试人员提供测试用例编写依据。

2.2 背景

目前，EMP离线协议规定的资源下载流程中，并未定义加解密和校验流程，离线资源在下载过程中使用明文传输，这使得含有用户信息的资源存在被窃取和篡改的风险。

3 任务概述

3.1 目标

当前版本的EMP离线资源协议并没有对资源进行加密传输，请求与响应均使用明文。并且服务端将所有离线资源存储在外网可以访问的路径下。这样会使得一些包含敏感信息的资源可以被轻易的获取到。因此，我们需要对EMP离线协议进行安全性优化，使其满足不同用户的安全需求。

3.2 系统特点

支持明文传输和密文传输，可基于EMP信道和HTTPS/SSL加密。

4 需求规定

4.1 软件功能说明

本次优化我们需要满足以下需求：

  1. 支持在服务端配置离线资源安全传输的方式：明文传输、EMP信道明文传输、EMP信道密文传输、HTTPS/SSL。
  2. 服务端根据配置在描述文件中增加传输方式字段，客户端根据该字段标识的传输方式下载资源。不符合规定的下载请求将被拒绝。
  3. 修改现有HTTP和TCP下载接口，使其支持四种传输方式。
  4. 修改服务端存储离线资源的路径，使资源不会被直接下载。
  5. 兼容旧版本客户端
