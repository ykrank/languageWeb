2 引言

2.1 目的

本文旨在说明可选资源单个文件下载的功能需求，使得设计人员和开发人员理解需要实现的具体功能，并为测试人员提供测试用例编写依据。

2.2 背景

可选资源均为插件包，当用户需要更新的资源较大或网络条件不好的情况下，更新整个可选插件包速度很慢，甚至很可能失败，用户体验较差。我们需要增加可选插件包中单个资源更新方法，使得用户可以每次更新较小的资源。

3 任务概述

3.1 目标

  • 可选插件更新实现增量更新方式
  • 实现获取可选资源更新进度的方法

4 需求规定

4.1 软件功能说明

  • 修改现有可选资源更新的实现方式，实现增量更新，只更新或删除需要更新的文件。
  • 修改现有Lua接口，增加回调方法参数，可以获取下载进度。
