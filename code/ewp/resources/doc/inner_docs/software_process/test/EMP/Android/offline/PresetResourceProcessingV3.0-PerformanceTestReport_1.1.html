5 测试结果分析

5.1 预置不加密，本地保存不加密

  • 耗时比较多的方法：（单位： ms）

    1. 离线3.0版本兼容系统升级时预置资源的处理： 25
    2. 预置资源描述信息DESC，new JSONObject（） 对象：20
    3. 服务器可选描述O字段，new JSONObject（）对象 ： 10
    4. H5资源描述信息， new JSONObject（） 对象：28

  • 数据保存：（单位： ms）

    1. 保存服务器可选（O字段）DESC：6
    2. 保存EWP必选及文件描述（文件个数：71）：4 平均：0.0563
    3. 保存EWP预置可选及文件描述（文件个数：41）：3 平均：0.0731
    4. 保存H5及文件描述（文件个数：714）：28 平均：0.0392

  • JSON解析：（单位： ms）

    1. 必选资源(文件个数：71)while循环解析json，耗时：0
    2. 可选资源（文件个数：41）while循环解析json，耗时：0
    3. H5资源（文件个数：714）while循环解析json，耗时：13

5.2 预置不加密，本地保存加密

  1. 预置资源处理完成需要时间：本地加密存储： 311 ms； 本地不加密存储: 151 ms 相差：160 ms

5.3 预置资源加密（非国密），客户端本地保存加密

  1. 读取 offline.json 文件耗费时间： 预置加密:43 ms ；预置不加密:2 ms。 相差：41ms
  2. 预置资源处理耗费时间：预置加密:347 ms ；预置不加密:311 ms。 相差：36ms

6 测试结论

  1. 尽量减少创建JSONObject对象次数，特别是json内容比较多时，但要保证数据的一致性。
  2. 比较大的数据要减少保存数据库的次数
  3. 能一起保存到数据库的，尽量一次性commit
  4. json解析耗时少，可以多次使用
  5. 不加密存储预置资源处理总时间151ms，加密存储预置资源处理总时间311ms。(151ms / 311ms) * 100 = 48.5 % 。可见本地加密存储时几乎多出一倍的时间。
  6. 预置资源的加密 与 不加密比较：仅在读取offline.json 文件时耗时相对比较多，对其他方法影响不大； 对整体预置资源处理影响也不是特别明显。
