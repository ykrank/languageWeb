1 测试内容

本次测试内容：ewp_http_client模块request性能测试。

2 测试环境

2.1 测试机器

ebank 服务器(四核): 系统：Ubuntu 12.04.5 LTS CPU：四核 Intel(R) Core(TM) i3 M 350 @ 2.27GHz 内存：4 G MySQL版本：5.1.73

simulator 服务器(四核)： 系统：CentOS release 6.5 (Final) CPU：四核 Intel(R) Xeon(R) CPU E5-2450 0 @ 2.10GHz 内存：2 G MySQL版本：5.1.73

simulator2 服务器(双核)： 系统：CentOS release 6.5 (Final) CPU：双核 Pentium(R) Dual-Core CPU E5700 @ 3.00GHz 内存：2 G MySQL版本：5.1.73

2.2 测试程序

erlang版本：R15B (erts-5.9) 64-bit ewp版本：5.2 ebank版本：5.2 simulator版本：5.2 mysql版本：5.1.73

3 测试方法

本次测试区别在ebank、simulator、simulator2这三台服务器的CPU差异，测试分两组：ebank->simulator、ebank->simulator2。

测试用例是使用ewp_http_client:request/1方法，通过模拟ebank的登录请求，performance_test:main/5性能测试函数不断的调用测试用例，使用cpu_usage:start/1统计函数统计时间，记录结果。 测试用例代码：

pf_simulator_test(IP, N_Pro, N_Times, LoopTime) ->
    UsageName = "Simulator_whole_" ++ integer_to_list(N_Pro) ++ "_"
                ++ integer_to_list(N_Times) ++ "_" ++ integer_to_list(LoopTime),
    cpu_usage:start_link(),

    cpu_usage:start(UsageName),
    performance_test:main(N_Pro, N_Times, fun simulator_test/1, {IP, N_Pro, N_Times, LoopTime}, LoopTime),
    cpu_usage:stop(UsageName),
    %% output the result of cpu_usage
    cpu_usage:dump().

simulator_test({IP, N_Pro, N_Times, LoopTime}) ->
    UsageName = "Simulator_controller_" ++ integer_to_list(N_Pro) ++ "_"
                ++ integer_to_list(N_Times) ++ "_" ++ integer_to_list(LoopTime),

    cpu_usage:start(UsageName),
    ewp_http_client:request({post, 
        {"http://" ++ IP ++ "/ebank", 
        [], "application/x-www-form-urlencoded;charset=utf-8", 
        <<"tranCode=login">>}, [], []}),
    cpu_usage:stop(UsageName),
    ok.

参数解释：

  • IP：simulator的IP地址。
  • N_Pro：并发数。
  • N_Times：执行测试函数的次数。
  • LoopTime：循环次数。 备注：测试函数的总执行次数为 LoopTime × N_Times。

  • 根据表5，随着并发不断加大，TPS也不断升高，最终稳定在1900左右。
  • 根据表6、7，simulator2服务器的%id空闲CPU使用率已接近0，即该服务器的CPU已达到瓶颈。
  • 根据表7、8，由于simulator2服务器CPU的瓶颈，使得网络IO基本稳定。
  • 根据表1、5，可以看出表1的request时间要长于表5的时间，是因为表1的ebank服务器CPU使用已达瓶颈，影响了单次响应时间。

5 总结

  • ebank服务器在处理高并发的http请求时，会大量消耗CPU的性能，在CPU达到四核时，至少可以满足2100并发的正常处理。
  • TPS会随着CPU性能的提升而得到提高，单次http的request时间，ebank服务器的CPU越繁忙，request时间会越长。
