1 测试内容

本次测试内容:

  • 防重复提交功能本身的运行效率

  • 防重复提交功能对现有业务运行效率影响

2 测试环境

2.1 测试机器

本地虚拟机(双核)： 系统：CentOS release 5.4 (Final) CPU：Intel(R) Core(TM) i3-2350M CPU @ 2.30GHz 内存：4 G MySQL版本：5.5.31

simulator 服务器(四核)： 系统：CentOS release 6.5 (Final) CPU：四核 Intel(R) Xeon(R) CPU E5-2450 0 @ 2.10GHz 内存：2 G MySQL版本：5.1.73

2.2 测试程序

erlang版本：R15B (erts-5.9) 64-bit ewp版本：5.2 ebank版本：5.2

3 测试方法

  • 防重复提交功能本身的运行效率性能测试在本地虚拟机和simlulator分别进行测试，在不同系统配置下，对本身的运行效率的影响，通过性能测试函数performance_test:main/5，对防重复提交功能测试用例函数trans1/0的调用，main/5的最后一个参数是循环次数，设置为5，结果取5次数据中的中间值，测试函数是下面代码的test_trans/2函数。

  • 对现有业务运行效率影响测试是在本地虚拟机上，模拟业务请求，在ebank业务逻辑中添加防重复提交的代码，通过cpu_usage:start/1，cpu_usage:stop/1统计防重复提交功能逻辑消耗的时间和请求消耗的时间，同样通过性能测试函数performance_test:main/5对测试用例函数trans2/0的调用，main/5的最后一个参数是循环次数，设置为1，测试函数是下面的test_trans_eff/2函数。

参数解释：

  • M：并发数。
  • N：执行测试函数的次数。

4 测试场景

4.1 本身效率测试

测试条件：测试用例调用次数相同，并发数不同的情况下，进行数据统计。
