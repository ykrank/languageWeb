根据界面需求,需要实现控件滑动的功能. 即试图根据手指的移动同步移动.

参考 html5的 touchevent 的对象结构. 我们需要三个类来保持 touch 事件的数据.

1 Touch

一个 Touch 对象代表一个触发点.

interface Touch {
    readonly    attribute number        identifier;
    readonly    attribute RYTControl    target;
    readonly    attribute number        clientX;
    readonly    attribute number        clientY;
    readonly    attribute number        pageX;
    readonly    attribute number        pageY;
    readonly    attribute number        viewX;
    readonly    attribute number        viewY;
    readonly    attribute number        contentX;
    readonly    attribute number        contentY;
    readonly    attribute integer  tapCount;
  };

identifier of type number, readonly An identification number for each touch point. When a touch point becomes active, it must be assigned an identifier that is distinct from any other active touch point. While the touch point remains active, all events that refer to it must assign it the same identifier.

target of type RYTControl, readonly The RYTControl on which the touch point started when it was first placed on the surface, even if the touch point has since moved outside the interactive area of that element.

clientX of type number, readonly The horizontal coordinate of point relative to the body view in pixels, excluding any scroll offset

clientY of type number, readonly The vertical coordinate of point relative to the body view in pixels, excluding any scroll offset

pageX of type number, readonly The horizontal coordinate of point relative to the body view in pixels, including any scroll offset

pageY of type number, readonly The vertical coordinate of point relative to the body view in pixels, including any scroll offset

viewX of type number, readonly The horizontal coordinate of point relative to the target view in pixels, excluding any scroll offset

viewY of type number, readonly The vertical coordinate of point relative to the target view in pixels, excluding any scroll offset

contentX of type number, readonly The horizontal coordinate of point relative to the body view in pixels, including any scroll offset

contentY of type number, readonly The vertical coordinate of point relative to the body view in pixels, including any scroll offset

tapCount of type integer, readonly The number of times the finger was tapped for this given touch.

2 TouchList

touchList 是 touch 对象的集合, 对于有多点触摸的情况,touch 会有多个.
通过 lua table 的方式实现此列表.

3 TouchEvent

interface TouchEvent : UIEvent {
      readonly    attribute TouchList touches;
      readonly    attribute TouchList targetTouches;
  };

touches of type TouchList, readonly a list of Touches for every point of contact currently touching the surface.

targetTouches of type TouchList, readonly a list of Touches for every point of contact that is touching the surface and started on the element that is the target of the current event.

4 Listerner

touchstart,touchend,touchmove,touchcancel.
实例:
`document.getElementById('touchable').addEventListener('touchend', function(ev));`

监听方法包含参数 ev, 参数类型 TouchEvent.

使用方式:
  1. 通过addEventListener方法为控件设置监听. 参数包括监听类型,监听方法.
  2. 当对应的控件接收到点击事件后,回调监听方法,并传递 TouchEvent 参数.
  3. 脚本通过 TouchEvent 对象可以获取 所有touch 点得信息.从而进行相应的操作.

5 Lua接口

6 element:addEventListener();

  • Description:
    设置元素的监听函数,并指定监听类型. 一个元素只能设置一个监听函数.
  • Syntax:
    • element:addEventListener(eventType,function).
  • Parameters:
    1. eventType(required)
      • 类型: String.
      • 说明: 指定监听事件类型.值包括touchstart,touchend,touchmove,touchcancel.
    2. function(required)
      • 类型: function.
      • 说明: 监听事件的回调函数.
  • Return:
    无

7 element:removeEventListener();

  • Description:
    移除当前控件指定事件类型的监听函数.
  • Syntax:
    • element:removeEventListener(eventType).
  • Parameters:
    1. eventType(required)
      • 类型: String.
      • 说明: 指定监听事件类型.值包括touchstart,touchend,touchmove,touchcancel.
  • Return:
    无

8 使用实例

依控件随手指滑动为例:
设置监听函数

function touchstartListener(event){
    touch = event.targetTouches.item(0);// 获取一个 touch 对象
    target = touch.target;//获取被点击的控件.
    preX = touch.pageX;
    preY = touch.pageY;
    currentX = target.getStyleByName('x');
    currentY = target.getStyleByName('y');
}
function touchmoveListener(event){
    touch = event.targetTouches.item(0);// 获取一个 touch 对象
    target = touch.target;//获取被点击的控件.
    x = touch.pageX;
    y = touch.pageY;
    distenceX = x - preX;
    distenceY = y - preY;
    preX = touch.pageX;
    preY = touch.pageY;
    target.setStyleByName('x',currentX + distenceX);
    target.setStyleByName('y',currentY + distenceY);
}

document.getElementById('touchable').addEventListener('touchstart', touchstartListener);
document.getElementById('touchable').addEventListener('touchmove', touchmoveListener);

9 FAQ:

9.1 事件响应顺序

目前的事件包括手势, touch 事件, onclick 事件. 规定:

  1. 手势的优先级最高.当控件增加了手势监听后,需要在手势响应失败后才能发起 touch 事件的监听. 如果手势成立,则执行手势的回调,不再执行 touch 事件和 onclick 事件.
  2. 当控件同时设置了onclick和touchend事件时，先执行touchend监听，再执行onclick事件。

注：为了防止onclick与end事件的冲突，建议使用其中一个来完成点击事件。

9.2 线程问题

emp的lua脚本是在lua 线程中执行,但是touch事件的响应是在主线程中.此时touch事件监听的回调需要频繁的在主线程和lua线程中切换.这会严重影响执行的效率. 同时为了防止lua的执行影响主线程的响应,我们将lua的执行做成了异步.对于时序要求不高的操作,这样是没有问题的. 但是在touch响应中,线程的异步导致主线程回调多次,lua线程才回调一次. 导致效果卡顿.

因此将touch 事件的处理移到主线程执行.在测试中发现,仅仅移到主线程虽然效果比在子线程要好很多,但是依然不理想.

由于touchmove的回调调用频繁,而每次创建监听回调的参数比较耗时. 因此建议公用这个参数, 及这个回调参数只创建一次,之后的回调里仅仅更新其中的值,而不需要创建lua table.

为了防止多线程同时操作lua状态机的情况,在touch事件相应过程中,需要将lua线程暂停.当touch结束后再执行lua线程的操作.

参考资料: http://www.w3.org/TR/touch-events/
