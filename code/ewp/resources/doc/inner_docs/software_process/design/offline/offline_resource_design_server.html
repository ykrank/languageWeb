由于在EMP5.2版本对离线协议进行了一次优化，部分规范和设计进行了调整和修改，因此引入了离线资源版本号来标识新旧版本。EMP5.2之前的旧离线协议版本号定义为0，EMP5.2及以后的新版本号定义为1。

1 资源文件上传

资源文件上传可通过管理后台和ewp_ctrl脚本进行上传。上传后，资源文件将保存在docroot/resources的子目录下。保存规则为：

  1. 如未指定平台和分辨率，则对应文件将保存在common/extension/子目录下。
  2. 如只指定了平台，则文件将保存在platform/common/extension/子目录下。
  3. 如指定了平台和分辨率，则文件将保存在platform/resolution/extension/子目录下。

离线资源的加密标记、下载类型、备注等各属性值会保存在同目录下的property文件中，命名规则为“资源文件名.property”。这些文件内容在生成描述文件时被读取出来，从而获取各资源的属性值。

保存资源文件后，EWP会遍历所有资源目录，使用SHA1算法计算每个文件hash值（包括插件zip包中的各个文件），并按上述格式重新生成资源描述文件。如在不同子目录下存在同名的资源文件，在生成指定平台和分辨率的描述文件时会按照以下优先级生成文件描述：

platform/resolution子目录 > platform/common子目录 > common子目录

2 描述文件管理

我们需要在服务端保存针对指定平台和分辨率的资源文件的描述信息，这些描述信息以文件方式存储在离线资源存储目录的desc子目录下，以app-platform-resolution-version.desc命名。各版本格式如下：

  • 旧版本格式：
{
    "resources": {
        "home.html": {
            "download_type":0, //下载类型，0为必须，1为可选。默认为0
            "encrypt":0, //加密标记，0为不加密，1为加密。默认为0
            "rev": "ad4d26cc1dcc7210b6f7…", //文件哈希值
            "path": "iphone/320*480/html" //文件存放路径
        },
        "home.zip": {
            "rev": "ad4d26cc1dcc7210b6f7…",
            "path": "common/css",
            "enc":0,
            "desc": { //插件资源包里的各资源描述
                "home/index.xhtml": "c4f3cffb7d68e63531c60e74611a2d2f569173e7",
                ......
                }
            }
        },
        ...
    }
}
  • 新版本格式：
{
    "hash": "c4f3cffb7d68e63531......", // 对必选资源描述计算出的哈希值
    "hash_opt": "c4f3cffb7d68e63531......", // 对可选资源描述计算出的哈希值
    "l": { //必选下载资源描述
        "home.html": {
            "e": 0, //加密标记，0为不加密，1为加密。默认为0
            "r": "ad4d26cc1dcc7210b6f7…", // 文件哈希值
            "p": "iphone/320*480/html", // 文件存放相对路径
            "u": 1//确保更新标记。1为确保，反之为0。默认为0
        },
        "home.zip": {
            "r": "ad4d26cc1dcc7210b6f7…",
            "p": "common/css",
            "e":0,
            "c": { //插件资源包里的各资源描述
                "home/index.xhtml": {
                    "r" = "c4f3cffb7d68e63531c60e74611a2d2f569173e7",
                    "u": 1
                },
                ......
            }
        },
        ...
    },
    "o": {
        //可选下载资源描述，格式与必选资源相同
    }
}

3 资源更新检测

在离线协议1版本中加入了描述文件hash值比对接口ota/resource_hash，用以比对可选和必选资源描述文件是否有修改。如有修改，客户端须再次请求资源更新检测接口ota/resource_update来获取已更新的资源列表。在0版本中，客户端需要每次都访问ota/resource_update接口。

在ota/resource_hash处理逻辑中，服务端会根据客户端上送的平台和分辨率获取到描述文件。如获取失败返回0，否则获取描述文件中hash和hash_opt值与客户端上送的哈希值进行比对，规则为：

  1. 全部相同返回0。
  2. 只有可选资源哈希值相同返回1。
  3. 只有必选资源哈希值相同返回2。
  4. 都不相同返回3。

获取描述文件规则为：

  1. 如果存在相同平台和分辨率的描述文件，则返回此文件。
  2. 如果存在相同平台，则返回分辨率相近的描述文件。分辨率比较算法为宽度差的平方与高度差平方之和。
  3. 否则描述文件获取失败。

为了简化交互流程，resource_hash校验功能也支持在握手（exchange/handshake接口）时完成。客户端需在url中添加platform和resolution参数，并在post body追加resource_hash和resource_hash_opt消息，消息格式为Type(1 byte) + Length(4 bytes) + Hash(Length bytes)。resource_hash和resource_hash_opt的Type值分别为0x0C、0x0D。如未读取到resource_hash和resource_hash_opt消息，则服务端不会执行资源描述哈希值比对流程。如读取到，则服务端会根据平台和分辨率获取本地描述比对哈希值，并在响应中追加resource_hash_res消息，格式为Type(1 byte) + Length(4 bytes) + Result(Length bytes)。Type值为0x0E，Result为字符0、1、2、3加密后的内容。若比对失败，则Result为-1。

在ota/resource_update处理逻辑中，服务端会根据客户端上送的平台和分辨率获取到描述文件。如获取失败，则返回更新标识-1。否则解析客户端上送的必选资源描述文件内容，将服务端所有资源的哈希值与客户端相同资源的哈希值进行比对，比对结果返回给客户端。

返回结果一定会包含更新标识，可能会包含主机地址、下载列表、可选资源列表和删除列表，0版本时会将服务端描述文件一同返回给，1版本时会将tcp下载端口返回。更新标识为-1时，表示无更新。为0时提示用户非强制更新。为1时提示用户强制更新。更新标识0和1可在服务端管理后台配置，默认为0。当响应只有删除列表时，更新标识为1。

3.1 离线协议1版本资源比对规则

1版本中，客户端会将ota/resource_hash接口的返回值作为ota/resource_update接口的hash_res参数上送给服务端。如果hash_res为1，则服务端只比对必选资源。如果hash_res为2，则服务端直接生成可选资源列表。如果hash_res为3，则服务端须比对必选资源并生成可选资源列表。必选资源比对规则为：

  1. 如资源只存在在服务端，则将该资源加入到下载列表中。
  2. 如资源只存在在客户端，则将该资源加入删除列表中。
  3. 如资源哈希值相同，则不做处理。
  4. 如普通资源哈希值不同，则将该资源加入到下载列表中。

  5. 如插件资源包哈希值不同，则比对插件资源包中每个资源的哈希值，并将插件包的下载列表和删除列表加入到对应列表中。插件包资源的比对规则为：

    1. 如资源只存在在服务端的插件包中，则将该资源加入到该插件包的下载列表中。
    2. 如资源只存在在客户端的插件包中，则将该资源加入到该插件包的删除列表中。
    3. 如插件包中某一资源哈希值不同，则将该资源加入到该插件包的下载列表中。

3.2 离线协议0版本资源比对规则

0版本中，服务端会遍历本地描述文件，将所有只存在在客户端的资源加入到删除列表中，将所有可选资源加入到可选列表中，将哈希值不同或只存在在服务端的资源加入到下载列表中。

4 预置资源下载

管理后台离线存储模块提供了预置资源包下载功能。预置资源包一定会包含当前服务端所有的必选资源、预置资源描述文件，并会根据是否下载可选资源参数值来预置可选资源。 在0版本中，预置资源描述文件即服务端资源描述文件。在1版本中，预置资源描述文件根据服务端资源描述文件生成。一定会包含必选资源列表。如服务端存在可选资源，还会包含可选资源列表，并根据是否下载可选资源参数值来生成已预置的可选资源列表。具体格式为：

{
    "l" : //必选资源列表
    {
        "s" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
        "zip1.zip" : 
        {
            "r" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
            "e" : 1 //如果不需要加密存储,则没有此字段
            "c" : 
            {
                "file1" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7" //file1为相对于zip1的相对路径 
            }
        }
        "file1" : 
        {
            "r" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
            "e" : 1 //如果不需要加密存储,则没有此字段
        }
    }
    "p" : //已预置可选资源列表
    {
        "zip2.zip" : 
        {
            "r" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
            "e" : 1 //如果不需要加密存储,则没有此字段
            "c" : 
            {
                "file1" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7" //file1为相对于zip1的相对路径 
            }
        }
    }
    "o" : //可选资源列表
    {
        "t":4060, //tcp下载端口
        "s" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
        "dir1" :  //如: ebank/resources/common/zip
        {
            "zip1.zip" : //zip包中部分文件的更新 如: account.zip
            {
                "r" : "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
                "e" : 1 //1,加密, 0,不加密 默认为0,且如果为默认,则省略此字段  
                "c" : //需要下载文件的列表
                {
                    "file1" :   //file1为相对于zip1的相对路径,如: account/xhtml/index.xml       
                    {
                        "r" = "c4f3cffb7d68e63531c60e74611a2d2f569173e7"
                        "u": 1  //1,确保更新, 0,非确保更新 默认为0,且如果为默认,则省略此字段
                    }   
                }
            }
        }
    }
}

5 TCP下载

在离线协议1版本中增加了TCP下载功能，服务端新增了ewp_offline_server进程来实现此功能。ewp_offline_server具有连接管理、资源下载和资源缓存的功能。

5.1 连接管理

ewp_offline_server进程启动时会监听指定端口，然后另启进程去接收连接，接收到连接后变执行下载流程，同时再启一个进程接收连接。连接数与子进程数是一致的。ewp_offline_server进程本身不接收连接，而是管理连接数量。当连接数达到配置的最大值时，ewp_offline_server会关闭监听的socket，直到有子进程完成下载并关闭连接，ewp_offline_server将重新监听。

5.2 资源下载

子进程在收到客户端包后，会按指定格式对包进行解析，如果解析错误则返回错误码2并关闭连接。解析成功后，服务端会根据指定的路径读取文件。如果文件不存在，或文件不在docroot下，则返回错误码1并关闭连接。读取成功后，服务端将文件内容按指定格式封装并发送给客户端。正常下载流程中，服务端不会主动断开连接，须客户端主动断开连接。如果一段时间内没有接收到客户端包，服务端会将该连接断开。

5.3 资源缓存

为了提高下载效率，ewp_offline_server实现了资源缓存功能。如果开启缓存功能，ewp_offline_server会在某一资源首次下载时将文件内容缓存到ets内存表中，以文件路径为key。之后下载该资源时将从ets表中读取文件内容，避免读取磁盘，提高读取效率。

ewp_offline_server会启动一个子进程来定时（10 min）清理资源缓存，当某一资源一段时间没有被下载后，清理进程会将这条记录从ets表中清除。

资源缓存功能会提高单个文件的下载效率，但会占用内存资源。须根据实际情况选择是否开启此功能。

5.4 配置

为了对离线资源TCP下载进行配置，在ewp.conf文件中增加了offline配置项。其格式为：
