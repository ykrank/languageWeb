1 安装Jenkins

1.1 准备工作

  1. 安装jdk
  2. 安装tomcat
  3. 下载Jenkins

1.2 Windows安装Jenkins

  1. 将下载的jenkins.war放在D:\Program Files (x86)\apache-tomcat-7.0.50\webapps 目录下
  2. 重启tomcat服务
  3. 浏览器输入：http://localhost:8080/jenkins 显示如下图则安装正确：

1.3 Linux安装jenkins

  1. 将下载得到的jenkins.war复制到/usr/local/apache-tomcat-7.0.50/webapps
  2. 重启tomcat
  3. 打开localhost:8080/jenkins，显示如图：

1.4 Mac安装jenkins

  1. 当做tomcat服务的安装方式和linux的相同，请参考Linux安装jenkins
  2. 也可通过home brew安装jenkins，brew install jenkins
  3. 启动jenkins，查看页面

1.5 Jenkins升级

  1. war：下载新版的war文件，替换旧版本war文件，重启即可。
  2. 二进制：卸载旧版本，安装新版本即可。

Jenkins程序下载地址

注意：升级前，请测试该版本和你本地数据的兼容性。如何测试：将JENKINS_HOME拷贝一份到新的机器，用新版的程序启动。测试对应的插件和配置。

2 配置FTP

  1. 在jenkins的插件管理中选择安装FTP publisher plugin
  2. 进入系统管理 >> 系统设置页面，配置FTP
  3. 需要在项目构建的最后阶段调用ftp发布的方式配置

注意：

  • 不要选中那个skipping publishing选项，那个就直接忽略这个ftp操作了
  • use timesstamps 这个会将我们发布的时间作为目录名称自动创建

3 配置邮件服务

  1. 进入系统管理 >> 系统设置页面，配置email
  2. 在jenkins job里面配置邮件发送

5 类Unix系统从的节点配置

  1. 在类Unix节点生成SSH密钥对（中间多次询问输入，直接输入回车即可）
    zc-w1:~ test$ ssh-keygen -t rsa
  2. 将公钥添加到这台主机的 authorized_keys 文件中，并修改权限为600
    zc-w1:.ssh test$ cat id_rsa.pub >> authorized_keys
    zc-w1:.ssh test$ chmod 600 authorized_keys
  3. Jenkis的页面上，配置访问这个服务器的“Credential”，在Jenkins页面的左侧快捷栏中点击“Credentials”

    点击Credentials->add credentials

    在“Kind”里，我们选择“SSH Username with private key”；“Username”这里输入登录上面服务器的用户名“jenkins”；“Private Key”这里选择“Enter directly”，输入内容就是上面步骤里创建的 id_rsa 文件内容。
  4. 在节点安装java
  5. 一切就绪后，在jenkins页面创建从节点，点击“系统管理”->“管理节点”->“新建节点”，进入创建节点页面
    输入节点名，选择“Dumb Slave”
  6. 节点配置

    启动方法：选择“Launch slave agents on unix machines via SSH”；HOST：从节点IP地址；Credentials：选择刚创建的连接认证；JAVA_PATH：从节点java目录
  7. 连接测试

    配置成功

    6 Jenkins配置ldap

  8. 在jenkins的插件管理中选择安装，LDAP Plugin.
  9. 进入系统管理--Configure Global Security页面，选择启用安全选项
  10. 安全域选择ldap，配置ldap信息
  11. 授权策略，当前选择为项目矩阵授权策略

7 Jenkins构建项目

  1. 启动jenkins服务，进入主页面点击新建按钮
  2. 输入Item名称，选择构建一个自由风格的软件项目，点击OK
  3. 输入项目名称和描述
  4. 配置是否丢弃旧的构建，考虑到磁盘资源的占用现在配置为保持最大的构建个数为10
  5. 选择当前job执行在那一台节点机器上
  6. 源码管理选择Mercurial(Mercurial插件jenkins默认没有安装， 需要去插件管理中心下载安装)。安装完成后需要在系统设置Mercurial安装目录, Installation directory配置本机的Mercurial安装目录，图示为Windows的配置：
  7. Revision Type是选择使用Branch或Tags拉取代码
  8. 输入 Repository URL（代码存放的url地址）、Credentials（输入用户名和密码）、Branch（代码中的分支）注意：分支或tags版本错误后，项目在构建的时候会报错（abort: unknown revision 'default'!）
  9. 构建触发器分为四种：
    • 在其他项目构建完成后才执行构建：依赖于其他构建项目执行
    • 触发远程构建 (例如,使用脚本)
    • Build periodically：按照特定的时间任务进行构建，任务格式采用的时cron表达式
    • Poll SCM：定期检查版本库是否有更新，如果有更新则触发Jenkins进行构建
      产品现在选择Build periodically输入H 1 1-31 * *（每天凌晨一点到两点之间随机分钟进行构建）
  10. 在构建项目页面点击“立即构建”按钮，构建成功后显示如图
  11. 构建状态分为四种：
    • 红色=失败
    • 蓝色=成功
    • 灰色=未执行完成
    • 黄色=不稳定的构建，构建过程中有可忽略的错误

8 获取构建程序

  1. 在jenkins job配置页面查看ftp配置信息
  2. 登录ftp服务器(可使用ssh命令和ssh相关的客户端工具) 
    ➜  ~  ssh rytongtest@192.168.64.70
rytongtest@192.168.64.70's password:
Last login: Fri Aug 22 09:47:00 2014 from 192.168.65.51
-bash-3.2$
  3. 连接ftp成功后，进入到build目录（build目录是存放ftp相关文件的路径） 
    -bash-3.2$ cd build/
-bash-3.2$ ls
android_deploy       androidtestupload         iphone_test     ryt-iphone
android_rebuild      iphone_auto_test          qt_deploy       ryt-winphone
android_rebuild_5.2  iphone_deploy             qt_rebuild      winphone_deploy
android_rebuild_5.3  iphone_deploy_beforeTest  qt_rebuild_dev
  4. build目录下的路径和jenkins配置页面的Destination一致（本例中就是ryt-iphone）
  5. Destination目录下以时间作为目录（精确到秒），找到需要的时间目录
  6. 依次进入目录即可找到需要下载的安装包,例如 
    build/ryt-iphone/2014-08-22_01-32-06/DailyBuild/output/AppStore
