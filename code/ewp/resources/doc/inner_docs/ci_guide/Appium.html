1.3 Android依赖环境

mac：

  1. 确保你已经安装了 Android SDK installed。
  2. 确保你安装的 Android SDK API 大于等于 17。你可以运行 Android SDK 管理器（android） 然后在额外的包里选择你要安装的 API。
  3. 确保你配置好了$ANDROID_HOME，并指向你的 Android SDK 目录。比如你的把 Android SDK 解压到 /usr/local/adt/， 那你就要将如下添加到你的 .bashrc 或 .zshrc 或 .bash_profile 等 shell 配置文件中去: export ANDROID_HOME=“/usr/local/adt/sdk”

windows：

  1. 安装nodejs 0.8版本及以上, 通过官方的安装程序来安装。
  2. 安装android的sdk包，(http://developer.android.com/sdk/index.html), 运行依赖sdk中的'android'工具。并确保你安装了Level17或以上的版本api。设置ANDROID_HOME系统变量为你的Android SDK路径，并把tools platform-tools两个目录加入到系统的Path路径里。
  3. 安装java的JDK，并设置JAVA_HOME 变量为你的JDK目录。

1.4 iOS依赖环境

mac：

  1. Appium 需要 Mac OS X 10.7以上版本
  2. 确保 Xcode 和 iOS SDK 都已经安装好了。 （当前 Appium 支持 Xcode 4.6.3/iOS 6.1 和 Xcode 5/iOS 7.0。 注意不推荐在基于 Xcode 5 下且低于 7.0 的 iOS 版本进行测试。）
  3. 你需要授权 iOS 模拟器的使用。如果你是通过 NPM 安装的 Appium，那么你可以运行 sudo authorize_ios （authorize_ios）是来自 Appium npm 包里的一个二进制执行文件。如果你是从源代码运行 Appium，那么你可以简单的使用 sudo grunt authorize。如果你使用Appium.app, 那你只要用界面来操作。

2 编写脚本

2.1 脚本示例

var wd = require("wd");
require('colors');

var desired = {
    "appium-version": "1.2",
    platformName: "iOS",
    platformVersion: "7.1",
    deviceName: "iPhone",
    app: "/Users/root/work/ipaFile/Release-iphonesimulator/APP.app",
};

var browser = wd.promiseChainRemote("0.0.0.0", 4723);

browser.on('status', function(info) {
  console.log(info.cyan);
});
browser.on('command', function(meth, path, data) {
  console.log(' > ' + meth.yellow, path.grey, data || '');
});

browser
    .init(desired)
    .waitForElementByName("访问").click()
    .waitForElementByName("基础控件(BaseController)").click()
    .quit();
  • require用来导入第三方库
  • desired是需要测试的应用和配置信息
  • wd.promiseChainRemote("0.0.0.0", 4723);是监听一个本地端口4723
  • browser是本地服务端对象
  • browser.init(desired)初始化
  • waitForElementByName("访问").click()点击当前页面的访问按钮
  • quit()关闭本地服务，退出模拟器

2.3 常用API

  • init(desired, cb) -> cb(err, sessionID, capabilities)初始化本地服务，用来发送客户端事件
  • quit(cb) -> cb(err) 删除本地服务
  • waitForElementByName(value, asserter, timeout, pollFreq, cb) -> cb(err, el) timeout时间内在页面获取名称为value的控件
  • waitForElementByXPath(value, asserter, timeout, pollFreq, cb) -> cb(err, el) timeout时间内在页面获取xpath为value的控件
  • element.elementByName(value, cb) -> cb(err, element) 在页面获取名称为value的控件
  • element.click(cb) -> cb(err) 发送点击事件

2.5 Android中文输入

android的测试框架Uiautomator不支持中文输入。
github上有一个开源的utf7ime输入法，可以实现中文输入，英文输入，中英文混合输入。简单来说，支持输入任何unicode编码的字符。 使用方法：

  • 打包下载 https://github.com/sumio/uiautomator-unicode-input-helper
  • Utf7Ime导入至eclipse
  • 使用eclipse导出apk并安装至手机或模拟器，并设置成默认输入法
  • 在脚本中直接使用send.keys()方法输入中文

3 测试框架

3.1 mocha

Mocha是nodejs使用的最广泛的测试框架，能很好的支持异步、同步测试风格，同时支持多种方式导出测试结果。

3.2 mocha用例结构

示例代码：

fs = require('fs');
describe('File', function(){
    describe('#readFile()', function(){
        it('should read test.ls without error', function(done){
            fs.readFile('test.ls', function(err){
                if (err) throw err;
                done();
            });
        })
    })
})
  • describe (moduleName, testDetails) 由上述代码可看出，describe是可以嵌套的，比如上述代码嵌套的两个describe就可以理解成测试人员希望测试Array模块下的#indexOf() 子模块。 module_name关键是要让人读明白就好。
  • it (info, function) 具体的测试语句会放在it的回调函数里，一般来说info字符串会写期望的正确输出的简要一句话文字说明。当该it block内的test failed的时候控制台就会把详细信息打印出来。 一般是从最外层的describe的module_name开始输出（可以理解成沿着路径或者递归链或者回调链），最后输出info，表示该期望的info内容没有被满足。一个it对应一个实际的test case
  • done () 按照瀑布流编程习惯，取名done是表示你回调的最深处，也就是结束写嵌套回调函数。但对于回调链来说done实际上意味着告诉mocha从此处开始测试，一层层回调回去。

3.3 Before && After

单元测试里经常会用到before和after。mocha同时还提供了beforeEach()和afterEach()。

var wd = require('wd');
var path = require('path');
var chai = require('chai');
chai.should();
require('colors');
var init = require('../../../lib/config');
var cF = require('../../../lib/checkFunc');

var timeout = init.data.timeout;
var images = init.data.imagesPath('iPhone', 'BaseController');
cF.checkPath(images);

describe('iOS BaseController link test >', function() {
  this.timeout(timeout);
  var browser = wd.promiseChainRemote(init.data.url, init.data.port);
  //print log
  // init.printLog(browser);

  beforeEach(function(done) {
    browser
      .init(init.device.iphone.Simulator71)
      .sleep(2000)
      .waitForElementByName('访问', timeout).click()
      .waitForElementByName('基础控件(BaseController)', timeout).click()
      .waitForElementByName('link_TEST', timeout).click()
      .nodeify(done);
  });

  afterEach(function(done) {
    browser
      .quit()
      .nodeify(done);
  });

  it('email test', function(done) {
    browser
      .waitForElementByName('客服邮件', timeout).click()
        .takeScreenshot()
        .saveScreenshot(images + 'emailTest')
      .catch(function (err) {
        console.log(err);
        throw err;
      })
      .nodeify(done);
  });
});
