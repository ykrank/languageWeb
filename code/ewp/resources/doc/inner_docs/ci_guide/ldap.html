1 安装LDAP

  1. 安装Berkeley DB
    export CPPFLAGS="-I/usr/local/berkeleydb/include"
export LDFLAGS="-L/usr/local/berkeleydb/lib"

    tar xvf db-6.0.20.gz
cd db-6.0.20
cd build_unix
../dist/configure --prefix=/usr/local/berkeleydb --enable-cxxmake
make install
    参考资源
    • 安装Berkeley DB
    • openLDAP/BerkeleyDB安装
  2. 安装LDAP服务
    wget ftp://ftp.openldap.org/pub/OpenLDAP/openldap-release/openldap-2.4.39.tgz
tar zxvf openldap-2.4.39.tgz
cd openldap-2.4.39
./configure
make depend
make
su root -c 'make install'
    官方文档

2 配置LDAP

vi /usr/local/etc/openldap/slapd.conf

database bdb
suffix "dc=rytong,dc=com"
rootdn "cn=Manager,dc=rytong,dc=com"
rootpw secret
directory /usr/local/var/openldap-data

启动

/usr/local/libexec/slapd
ldapsearch -x -b '' -s base '(objectclass=*)' namingContexts #检测，启动是否成功

重新启动LDAP

ps aux | grep slapd
kill <PID>

新建一个LDIF文件, 注意行末不能有空格

dn: dc=rytong,dc=com
objectclass: dcObject
objectclass: organization
o: emp
dc: rytong

dn: cn=Manager,dc=rytong,dc=com
objectclass: organizationalRole
cn: Manager

ldapadd -x -D "cn=Manager,dc=rytong,dc=com" -W -f example.ldif
检查添加是否成功
ldapsearch -x -b 'dc=rytong,dc=com' '(objectclass=*)'

3 LDAP客户端

  • ldap browser, CS模式，只有windows版
  • phpldapadmin, BS模式
  • Apache Directory Studio, CS版模式

这里我们在服务器上安装phpldapadmin

3.0.1 Yum安装方式：

  1. epel安装：
    rpm --import http://ftp.riken.jp/Linux/Fedora/epel/RPM-GPG-KEY-EPEL-6
rpm -ivh http://dl.fedoraproject.org/pub/epel/6/x86_64/epel-release-6-5.noarch.rpm
  2. 安装与配置httpd和php
    httpd安装配置:安装包管理工具的可以直接通过yum或者apt-get安装，例如：yum install httpd
    php安装配置
  3. 安装与配置phpldapadmin yum install phpldapadmin vi /etc/httpd/conf.d/phpldapadmin.conf 修改Allow from属性 Allow from 127.0.0.1 192.168.0.0/16 # IP address you allow
  4. 重启apache service httpd restart 访问phpldapadmin：http://IP/ldapadmin 登陆：cn=root,dc=abc,dc=com

3.0.2 源码安装方式：

  1. 安装与配置httpd和php：
    httpd安装配置:安装包管理工具的可以直接通过yum或者apt-get安装，例如：yum install httpd
    php安装配置
  2. 安装phpldapadmin
    yum erase phpldapadmin
wget http://sourceforge.net/projects/phpldapadmin/files/phpldapadmin-php5/1.2.2/phpldapadmin-1.2.2.tgz/download
tar -zxvf phpldapadmin-1.2.2.tgz
mv phpldapadmin-1.2.2 /var/www/html/
ln -s phpldapadmin-1.2.2 phpldapadmin
  3. 配置phpldapadmin： cd /var/www/html/phpldapadmin/config cp config.php.example config.php 修改下列config.php选项，注意行首不要有空格：
    $servers->newServer('ldap_pla');
$servers->setValue('server','name','HDT Linux LDAP');
$servers->setValue('server','host','127.0.0.1');
$servers->setValue('server','port',389);
$servers->setValue('server','base',array('dc=abc,dc=com'));
$servers->setValue('login','auth_type','cookie');
$servers->setValue('login','bind_id','cn=root,dc=abc,dc=com');
$servers->setValue('login','bind_pass','123456');
$servers->setValue('server','tls',false);
    phpldapadmin的使用方法，可参考：http://zhumeng8337797.blog.163.com/blog/static/100768914201010994913503/

4 LDAP配置ssl

  1. 服务器端需要安装的软件 检查 OpenLDAP 和 OpenSSL 的组件是否已经安装，如果没有，需要先安装:
    openldap参考本文章节ldap安装
    openssl 安装
  2. 生成 SSL/TLS 服务器证书
    • 创建 CA:./CA –newca
    • 使用openssl生成证书:openssl req -new -nodes -keyout newreq.pem -out newreq.pem
    • CA 对证书请求签名:./CA.pl –sign
    • 证书文件：newreq.pem、newcert.pem（服务器证书）、cacert.pem（ca证书）
  3. 配置ldap服务 ldap.conf文件添加以下内容:
    TLS_CACERTDIR   /Users/rytong/Tool/certs
TLS_REQCERT allow
    slapd.conf文件添加以下内容：
    TLSCipherSuite HIGH:MEDIUM:+TLSv1:+SSLv2:+SSLv3
TLSCACertificateFile /Users/rytong/Tool/certs/cacert.pem
TLSCertificateFile /Users/rytong/Tool/certs/newcert.pem
TLSCertificateKeyFile /Users/rytong/Tool/certs/newreq.pem
TLSVerifyClient never
    TLSCipherSuite参数可用命令：openssl ciphers -v ALL查询当前openssl支持的版本
  4. 启动ldaps服务
    • 关闭ldap服务:kill -INT `cat /usr/local/var/run/slapd.pid`
    • 只启动ldaps服务:./slapd -h "ldaps:///"

4.1 Apache配置ldaps

当前使用版本apache版本为2.2 apache ldaps 修改http配置文件http.conf文件

  1. 添加信任的证书
    LDAPTrustedGlobalCert CA_BASE64 /Users/rytong/Tool/certs/cacert.pem
  2. 修改ldap访问连接为ldaps
    <Location /ldap-status>
Allow from yourdomain.example.com
AuthLDAPURL ldaps://127.0.0.1/dc=example,dc=com?uid?one
AuthzLDAPAuthoritative off
Require valid-user
</Location>
  3. 重新启动http服务器即可

4.2 Jenkins配置ldaps

  1. 添加keystore信任证书, $JAVA_HOME/bin/keytool -import -alias certname -file cert.cer -keystore $JAVA_HOME/jre/lib/security/cacerts
  2. 重启tomcat服务器
  3. 修改jenkins访问ldap连接为ldaps，在系统管理 >> Configure Global Security >> 访问控制 >> ldap
    服务器：ldaps://yourdomain.example.com

4.3 Sonar配置ldaps

  1. 添加keystore信任证书, $JAVA_HOME/bin/keytool -import -alias certname -file cert.cer -keystore $JAVA_HOME/jre/lib/security/cacerts
  2. 重启tomcat服务器
  3. 修改sonar配置文件sonar.properties的ldapURLldap.url=ldaps://yourdomain.example.com
  4. 重新启动sonar服务器sonar restart

4.4 Nexus配置ldaps

  1. 添加keystore信任证书, $JAVA_HOME/bin/keytool -import -alias certname -file cert.cer -keystore $JAVA_HOME/jre/lib/security/cacerts
  2. 重启nexus服务器
  3. Security > LDAP Configuration修改ldap配置
  4. 添加ldap验证，Administration > Server
  5. 保存配置后登录即可
