业务流程开发

  • 说明
  • 实现方法

1 说明

本章节将以标准银行为项目app，介绍如何实现一个简单的业务流程开发。要实现的效果如下图所示：

在页面一中输入用户id号，点击提交按钮跳转到页面二，页面二中显示id号以及所对应的用户名。

2 实现方法

在标准银行中新建一个频道，频道ID为"channel_test"，频道中包含两个页面，trancode分别为"mb0901"和"mb0902"，xhtml页面的完整代码如下：

页面一：channel_test_mb0901.xhtml
页面二：channel_test_mb0902.xhtml

在页面一中，输入用户id号后点击提交按钮，会触发business_req()函数，代码如下：

--请求下一页面
function business_req()
    local inputContent;
    --显示加载界面
    window:showContent("local:FullSLoading1.xml",loadingtag);
    local channelId = "channel_test";
    local trancode = "mb0902";
    --获取输入框内容
    local inputCrl = document:getElementsByName("test_input");
    if inputCrl and #inputCrl>0 then 
        inputContent = inputCrl[1]:getPropertyByName("value");
    end
    local params = {id=channelId,tranCode=trancode,userId=inputContent};
    invoke_trancode(channelId, trancode, params, callback_channel, {trancode=trancode,channelId=channelId});
end

business_req()函数调用标准银行中的invoke_trancode()函数请求下一页面的报文及数据，调用invoke_trancode()函数必传的参数为 channelId 和 trancode，本例中channelId为"channel_test"，trancode为页面二的业务流程标识"mb0902"。同时输入框中的输入值也作为参数传给了invoke_trancode()以用于后续获取相应数据。根据传入的channelId以及trancode参数值，invoke_trancode()会在src/channels/文件夹下寻找以channelId命名的erl文件，调用该文件中以trancode命名的函数，获取数据并返回。故我们需要在"channel_test.erl"文件中，添加"mb0902"函数，在该函数中添加如下代码:

'mb0902'(TranCode, Channel) ->

    Id = ?param("id", []),
    TranCode = ?param("tranCode", []),
    UserId = ?param("userId", []),
    ParameterList = [{id, Id},
                     {tranCode, TranCode},
                     {userId, UserId}],
    Res = ewp_adapter:invoke_procedure("simulator","mb0902",ParameterList),
    Response = cs_api:render("channel_test_mb0902", Res),
    Response.

"mb0902"函数首先将invoke_trancode()传入的params中的参数取出来，再调用ewp_adapter:invoke_procedure使用适配器。在适配器配置文件"adapter.conf"中，配置id为"mb0902"的procedure，如下所示：

%%channel_test
{procedure, [{id, "mb0902"},
             {adapter, "simulator"},
              {return_type, xml},
              {path, "ebank"},
              {use_sample_data,true},
              {gen_log, true},
             {data_sample, "public/test/mb0902.xml"},
              {parameters, [{"tranCode", "mb0902"}, {"accNo", "1234567"}]}]
}.

根据以上配置，id为"mb0902"的procedure会使用适配器"simulator"来获取数据，返回类型为xml，为简便，我们配置了data_sample选项，使用静态文件模拟从"simulator"中获取到的数据，"public/test/mb0902.xml"中的数据配置如下：

<root>
    <errorCode>000000</errorCode>
    <errorMessage></errorMessage>
    <info>
        <userId>01</userId>
        <userName>Jack</userName>
    </info>
</root>

其中，userId为输入的用户Id，userName为对应的用户名。该xml文件被返回给"mb0902"函数中赋给Res变量，再调用cs_api:render("channel_test_mb0902", Res)，将xml文件传入"channel_test_mb0902.cs"文件中做格式转换，返回json格式的数据。

"channel_test_mb0902.cs"文件需存放在public/cs/channels/channel_test文件夹下，编写代码如下：

{
    "return":{
        "error_code": "<?cs var:root.errorCode ?>",
        "error_msg": "<?cs var:root.errorMessage ?>",
        "info": 
            {"userId": "<?cs var:root.info.userId ?>",
            "userName": "<?cs var:root.info.userName ?>"
            }
    }
}

最后返回的json格式数据被存在globalTable全局变量中，在页面二代码中使用globalTable["mb0902"]即可取出，之后将json格式的数据转换为lua中的table格式，取出"userId"和"userName"赋值给界面中的label控件，代码如下：

<script type="text/x-lua">
    <![CDATA[
        local json_info = globalTable["mb0902"];
        local table_info = json:objectFromJSON(json_info);
        --用户信息
        local userInfo = table_info["return"]["info"];
        --用户id
        local userId = userInfo["userId"];
        --用户名
        local userName = userInfo["userName"];
        local label_crl = document:getElementsByName("test_label");
        if label_crl and #label_crl>0 then 
            label_crl[1]:setPropertyByName("text",userId);
            label_crl[2]:setPropertyByName("text",userName);
        end 
    ]]>
</script>

这样页面二就获取到了所需的数据并将其显示出来。关于invoke_trancode()函数内部所做的事可以参照标准银行具体代码。
