1 实现效果

进入界面时模拟ATM吐出账单动画将成功信息一点一点的往下显示出来，同时播放成功音乐。

2 实现方法

2.1 思路

  • 使用动画接口transition:translateY(object,y,duration)将结果div从上往下移动。
  • 使用audio:play()接口播放音频文件。

2.2 脚本

transition:setCurve(div_ctrl[1], curve.ease_inout);
transition:translateY(div_ctrl[1], 360, 2);
local au = audio:load("result.mp3");
audio:play(au,1);

3 注意事项

3.1 replace有动画

在进入此界面前，展示此页面调用的方法为：

history:add(responsedata["responseBody"]);
location:replace(responsedata["responseBody"]);

此时为直接调用replace方法没有使用任何展示动画，修改调用replace为有动画，然后看一下android平台和iphone平台效果。

修改后代码为：

location:replace(responsedata["responseBody"],transitionType.slideFromLeft);

关于replace接口的详细说明请参照参考手册。

android平台运行效果： 界面从左往右滑入，但是没有吐账单动画。

iphone平台运行效果： 界面先从左往右滑入，然后吐账单动画开始。

为什么android平台会得不到预期效果呢，客户端人员解释为： 页面跳转的replace有动画，在执行完replace动画后刷新body,导致body上其他控件的动画被截断。如果客户端修改动画实现机制会导致多个控件同时 做动画时出现内存溢出。所以需要界面和客户端同时修改此问题。

android修改：将replace的回调移到页面动画的onAnimationEnd中，保证replace页面动画执行完以后再执行replace的回调。

界面修改：将动画的执行移到replace的回调中。本页面的Lua方法是在页面渲染之前执行的，而replace的动画需要在页面渲染之后执行，如果模板仍然保持之前的写法，android没有办法处理。

针对界面修改可能会出现的问题为，页面动画方法在replace回调中能否找到。

先修改replace代码为：

    output_ani = {};

    local function replace_callback(params)
        local ani_name = params["ani_name"];
        output_ani[ani_name]();
    end;

    function callback(responsedata)
        if responsedata["responseCode"] == 1599 then
            location:replace(responsedata["responseBody"]);
            window:hide(loadingtag);
        elseif responsedata["responseCode"] == 200 then
            history:add(responsedata["responseBody"]);
            location:replace(responsedata["responseBody"],
                             transitionType.slideFromLeft,
                             replace_callback,
                             {ani_name = "ani_fun",name="huangmiaomiao"}
            );

        else
            ryt:alert("网络请求失败！");
        end;
    end;

修改吐账单动画为单独方法：

--动画
output_ani["ani_fun"] =
    function ()
        local div_ctrl = document:getElementsByName("div_trans_bill");
        if div_ctrl and #div_ctrl > 0 then
            transition:setCurve(div_ctrl[1], curve.ease_inout);
            transition:translateY(div_ctrl[1], 360, 2);
            local au = audio:load("result.mp3");
            audio:play(au,1);
        end;
    end;

iphone平台和android运行都符合预期效果。

所以在有界面跳转动画和界面初始化动画时请将动画在replace接口回调中实现。

需要先初始化将界面所需数据填充完成后再调用动画接口做动画。
