Table新布局算法

  • 说明
  • 示例
    • 显示顺序问题
    • 多行控件的垂直居中问题
  • 总结

1 说明

客户端修改了table的布局算法，支持了td中控件的多行显示，导致某些报文写法在新旧客户端上的显示效果有差异，故本章节通过示例说明使用新客户端后布局table需注意的问题。

2 示例

2.1 显示顺序问题

我们要实现一个如下图所示的标题栏：

有如下报文写法:

<?xml version="1.0" encoding="UTF-8" ?>
<content>
    <head>
        <style>
            .table_title{left:0px;top:0px;width:320px;position:fixed;}
            .tr_title{height:70px;background-image:url(title_bg.png);}
            .td_title{width:320px;}
            .label_title{left:70px;width:180px;text-align:center;font-size:18px;font-weight:bold;color:#FFFFFF;}
            .button_back{left:8px;width:54px;height:32px;background-image:url(back_but.png);}
            .button_main{right:8px;width:54px;height:32px;background-image:url(main_but.png);}
        </style>
    </head>
    <body>
        <table class="table_title" border="0">
            <tr class="tr_title" >
                <td class="td_title" align="center" valign="middle">
                    <label class="label_title">天气预报查询测试</label>
                    <input type="button" class="button_back" name="back_but" onclick="back_fun()"/>
                    <input type="button" class="button_main" name="main_but" onclick="main_page()"/>
                </td>
            </tr>
        </table>
    </body>
</content>

在旧版本的客户端中，table是不支持多行显示的，使用以上写法，td中的三个控件会在同一行根据left和right样式排列，得到了我们想要的效果。

而新客户端的显示效果如下：

由于新的table布局算法支持了td中控件的多行显示，td中的控件会先按照报文中出现的顺序排版，再根据left、right样式定位，即：label在第一位，两个button控件排在label后面，由于button定义了left和right样式，只能在label后另起一行显示。

将td中控件的顺序修改下就可以得到正确的显示效果，table的报文修改如下：

<table class="table_title" border="0">
    <tr class="tr_title" >
        <td class="td_title" align="center" valign="middle">
            <input type="button" class="button_back" name="back_but" onclick="back_fun()"/>
            <label class="label_title">天气预报查询测试</label>
            <input type="button" class="button_main" name="main_but" onclick="main_page()"/>
        </td>
    </tr>
</table>

2.2 多行控件的垂直居中问题

新客户端支持table对多行控件垂直居中对齐，控件的居中对齐方式为以中心线对齐。

以下报文用来验证多行控件垂直居中对齐的显示效果:

<?xml version="1.0" encoding="UTF-8" ?>
<content>
    <head>
        <style>
            .body{background-color:#FFFFFF;}
            .table_title{left:0px;top:0px;width:320px;position:fixed;background-color:#F0F0F0;}
            .tr_title{height:350px;}
            .td_title{width:320px;}
            .ctrl1{height:30px;width:60px;font-size:15px;text-align:center;background-color:#FFE4E1;}
            .ctrl2{height:40px;width:80px;font-size:18px;text-align:center;background-color:#FFE1FF;}
            .ctrl3{height:50px;width:100px;font-size:21px;text-align:center;background-color:#EED2EE;}
            .ctrl4{height:30px;width:60px;font-size:15px;text-align:center;background-color:#F0FFFF;}
            .ctrl5{height:40px;width:80px;font-size:18px;text-align:center;background-color:#B4EEB4;}
            .ctrl6{height:50px;width:100px;}
            .ctrl7{height:30px;width:70px;}
            .ctrl8{height:30px;width:150px;font-size:15px;}
            .ctrl9{height:40px;width:150px;font-size:10px;}
            .ctrl10{height:30px;width:80px;font-size:15px;}
        </style>
    </head>
    <body>
        <table class="table_title" border="0">
            <tr class="tr_title">
                <td class="td_title" align="center" valign="middle">
                    <label class="ctrl1">Label</label>
                    <input type="button" class="ctrl2" value="Button"/> 
                    <input type="text" class="ctrl3" value="Input"/> <br/><br/>

                    <select class="ctrl4"> 
                      <option>select1</option>
                      <option>select2</option>          
                    </select>                    
                    <a class="ctrl5">Link</a>
                    <img src="image_test.png" class="ctrl6"></img> <br/><br/>

                    <input type="switch" name="switch1" class="ctrl7" checked="checked">yes</input>
                    <input type="switch" name="switch1">no</input>
                    <input type="checkbox" class="ctrl8">Checkbox</input> <br/><br/>

                    <input type="segment" name="segment1" value="seg1" class="ctrl9">seg1</input>
                    <input type="segment" name="segment1" value="seg2">seg2</input>
                    <input type="segment" name="segment1" value="seg3">seg3</input>
                    <input type="radio" class="ctrl10">Radio</input>
                </td>
            </tr>
        </table>
    </body>
</content>

3 总结

  1. 在写table的报文时，要注意：td中多个控件在报文中的顺序应与其在界面中显示的顺序一致。
  2. 新的table布局算法已支持对多行控件进行垂直居中对齐，每一行控件都以其中心线为基准对齐。
