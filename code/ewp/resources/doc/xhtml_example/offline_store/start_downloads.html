发起多个下载动作

  • 效果图
  • 实现方法

1 效果图

如上图所示，我们给插件下载添加了进度条的显示效果，并且，不同插件可以同时发起下载动作，进度条显示的状态不同。本节讲述如何实现同时发起多个插件的下载动作。

2 实现方法

示例的完整代码链接为：start_downloads.xml

我们在上节代码的基础上，修改了download_channel()函数，以实现进度条状态改变的动画。

要实现多个插件下载动作的同步需要用到lua中的闭包思想。示例中show_progress()写在了download_channel()函数的内部，它可以访问download_channel()函数中定义的局部变量和参数，这些变量和参数被称为show_progress()函数的upvalues (如index变量)，show_progress()函数及其upvalues一起被称为一个闭包。闭包就类似于一个类，类可以被实例化为多个对象，同一个类的不同对象的属性之间互不影响。当多个下载动作被执行时，不同插件调用的show_progress()函数的upvalues之间不会相互影响，这样便能够实现独立并同步的下载。

show_progress()函数的实现思路如下：引用了一个变量index，代表动画步骤的索引，每次show_progress()函数被执行时index加1，根据index的值改变下载条按钮的背景图片，当show_progress()函数被重复执行时，图片不断变化，就实现了一个下载进度条在加载的动画。show_progress()函数的重复执行是由callback_download()函数中的计时器实现的。

要注意：

  1. 进度条显示完成后要停止计时器，并将index索引回置为0。
  2. 下载过程中要将下载条按钮设置为不可点击；下载完成后恢复为可点击，并改变其背景图片为初始的"下载"字样图片。
  3. 不同插件被同时下载时，它们分别引用的index变量互不影响，故能够同时显示进度条加载的动画。

插件下载或升级函数download_channel()的代码：

--[[点击下载或者升级]]--
function download_channel(channelid)
    --标记下载和升级: 0为下载，1为升级
    local update_flag;
    --标记进度条状态
    local index = 0;
    --下载进度动画timer
    local download_timer;
    --下载条按钮名称
    local download_name = "download_"..channelid;

    --显示进度条
    local function show_progress()
        local download_ctrl = document:getElementsByName(download_name);
        index = index + 1;
        if  index == 1 then
            download_ctrl[1]:setStyleByName("background-image","j20.png");
        elseif index == 2 then
            download_ctrl[1]:setStyleByName("background-image","j60.png");
        elseif  index == 3 then
            download_ctrl[1]:setStyleByName("background-image","j100.png");
        elseif index == 4 then
            download_ctrl[1]:setStyleByName("display","none");
            download_ctrl[1]:setStyleByName("background-image","offline-download.png");
            --设置下载条按钮可点击        
            changeProperty(download_name,"enable","true");
            --设置频道按钮为可点击
            changeProperty(channelid,"enable","true");
            if update_flag == 0 then
                window:alert("下载成功!");   
            elseif update_flag == 1 then
                --显示更新信息div
                changeStyle("update_div","display","block");
                changeProperty("update_mess","value",optDownloadServer[channelid..".zip"]);
            end;
            timer:stopTimer(download_timer);
            index = 0;
        end;
    end;

    --下载回调函数
    local function callback_download(result)
        if result == true then
            --下载成功后开始进度条的显示
            download_timer = timer:startTimer(1, true, show_progress);
        else
            if update_flag == 0 then 
                window:alert("下载失败！");
            else
                window:alert("升级失败!");
            end;
        end;
    end;

    --处在删除状态中时下载按钮不可用
    if flag == 1 then 
        window:alert("请先取消删除！");
        return;
    end;

    --根据背景图片判断为升级或者下载
    local bg_img = getStyle(download_name,"background-image");
    --下载
    if bg_img[1] == "offline-download.png" then
        update_flag = 0;
    --升级
    elseif bg_img[1] == "offline-upgrade.png" then
        update_flag  = 1;
    end;

    --设置下载条按钮不可点击        
    changeProperty(download_name,"enable","false");

    --下载和升级动作
    for key,value in pairs(optDownloadServer) do
        if key == channelid .. ".zip" then
            if offline_version == "0" then
                offline:downfile(key,value,callback_download);
            else
                offline:downOptionalFile(key,callback_download);
            end;
        end;
    end;            
end;
