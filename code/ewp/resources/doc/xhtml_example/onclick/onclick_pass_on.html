点击事件传递

1 主题

在HTML标准中，事件为自顶向下传递。为了了解我们的客户端是怎么进行事件的监听和响应的，我们写个简单例子进行试验一下。下面的例子只在iphone和android平台试验，其他平台有待验证。

1.1 例子试验

简单例子：

报文为能点击的控件的嵌套：

<body>
    <div class="div_content" name="div_content" border="0" onclick = "div_onclick()"> 
        <table class="table_content" border="0">
            <tr class="tr_content" name="tr_content" onclick="div_table_tr_onclick()">
                <td valign="middle">
                    <input class="but_tr" name="but_tr" type="button" value="测试点击" onclick="div_table_tr_button_onclick()" />
                    <div class="div_tr" name="div_tr" border="0" onclick="div_table_tr_div_onclick()">
                    </div>
                </td>
            </tr>
        </table>
    </div>
</body>

脚本为点击后改变当前点击控件的背景颜色：

    local flag1 = 1;
    local flag2 = 1;
    local flag3 = 1;
    local flag4 = 1;
    function div_onclick()
        local div_content = document:getElementsByName("div_content");
        if flag1 == 1 then
            div_content[1]:setStyleByName("background-color","#FF0000");
            flag1 = 0;
        else
            div_content[1]:setStyleByName("background-color","#7FBA00");
            flag1 = 1;                    
        end;
    end;

    function div_table_tr_onclick()
        local tr_content = document:getElementsByName("tr_content");
        if flag2 == 1 then
            tr_content[1]:setStyleByName("background-color","#FFFF00");
            flag2 = 0;
        else
            tr_content[1]:setStyleByName("background-color","#CCCCCC");
            flag2 = 1;                    
        end;                
    end

    function div_table_tr_button_onclick()
        local but_tr = document:getElementsByName("but_tr");
        if flag3 == 1 then
            but_tr[1]:setStyleByName("background-color","#00FF00");
            flag3 = 0;
        else
            but_tr[1]:setStyleByName("background-color","#787878");
            flag3 = 1;                    
        end;                 
    end

    function div_table_tr_div_onclick()
        local div_tr = document:getElementsByName("div_tr");
        if flag4 == 1 then
            div_tr[1]:setStyleByName("background-color","#00FFFF");
            flag4 = 0;
        else
            div_tr[1]:setStyleByName("background-color","#909090");
            flag4 = 1;                    
        end;                 
    end

根据上面的测试结果得到iphone平台的事件传递和HTML标准一致为自顶向下传递事件。

同时找了个android平台的手机按照同样的步骤测试了一下，得到的结果一样。

看看我们的这个例子，所有能点击的控件都有点击事件，如果碰到父控件有点击事件，但是子控件没有点击事件时会出现什么效果呢。

在body中增加一个没有点击事件的tr：

<tr class="tr_content1" >
    <td valign="middle">
        <div class="div_tr" name="div_tr" border="0">
            <label>无点击事件tr</label>
        </div>
    </td>
</tr>

在iphone平台点击此无点击事件tr发现任何效果。

在android平台点击此此无点击事件tr发现会调用div_onclick事件改变了div_content的背景颜色。

1.2 结论

根据上面的试验，我们得到结论：

  1. iphone平台和android平台在事件响应上有差异。
  2. 如果父控件和子控件都有点击事件，两个平台表现一致，即响应子控件事件。
  3. 如果父控件有点击事件，但是子控件没有点击事件，点击子控件，iphone平台不会将事件冒泡到父控件进而响应父控件事件。但是android平台会响应父控件事件。

2 解决方案

如果碰到这样的场景该如何解决此问题呢，目前想到的有两种解决方案。

2.1 添加事件

此种解决方案为在所有子控件上添加点击事件，此事件和父控件点击事件实现一样的内容。

2.2 增加遮罩层

此种解决方案为在父控件上覆盖一层接近透明的遮罩层，用此遮罩层的点击事件来实现父控件的点击事件。

在试验的例子中做简单的修改。

在body中增加一个div，同时去掉父控件div的点击事件。

修改body代码为：

<body>
    <div class="div_content" name="div_content" border="0" > 
        <table class="table_content" border="0">
            <tr class="tr_content" name="tr_content" onclick="div_table_tr_onclick()">
                <td valign="middle">
                    <input class="but_tr" name="but_tr" type="button" value="测试点击" onclick="div_table_tr_button_onclick()" />
                    <div class="div_tr" name="div_tr" border="0" onclick="div_table_tr_div_onclick()">
                    </div>
                </td>
            </tr>
            <tr class="tr_content1" >
                <td valign="middle">
                    <div class="div_tr" name="div_tr" border="0">
                        <label>无点击事件tr</label>
                    </div>
                </td>
            </tr>            
        </table>
    </div>
    <div class="alpha_style" name="div_content" border="0" onclick = "div_onclick()"> 
    </div>
</body>

在样式表中增加样式： .alpha_style{top:100px; width: 320px; height: 320px;display: block; filter:progid(alpha='0.1');}

此时只是无点击事件的tr在点击的时候会响应div_onclick事件，但是其他的控件由于被遮罩层挡住后无法响应自身的点击事件。

这个增加遮罩层方法适合遮罩层由其他事件控制其显隐的场景。
