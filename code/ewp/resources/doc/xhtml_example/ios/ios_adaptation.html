IOS75.3新方案适配用例

1 iOS 状态栏

iOS 7出现后,状态栏占用空间被解放出来,增加了应用程序的可显示区域。同时应用程序的显示区域不再与状态栏的显示与隐藏相关。

为了更好的支持这种效果,并使项目能够灵活的控制显示的效果。我们把屏幕高度与状态栏的显隐分离。并提供接口由项目自定义状态栏所需要占用的空间。

这样可以支持iOS 6, iOS 7 两种状态栏效果。

详细内容可参考文档：https://dev.rytong.me/emp5.3/ert_develop/client_dev/emp_config/ert/ios/ios_statusBar.html

2 引发的问题

iOS状态栏占用空间被解放出来以后，为了报文适配其他平台，起初由于用户需求的单一化我们对于IOS7导航栏的 适配是客户端自动为状态栏加上一个纯色背景条。后来，随着用户需求逐步多样化，比如用户要求从坐标点{0,0}开始往下为 一张图片，使用当时的产品就不能满足此需求。所以经过深入细致的讨论与多次试验论证，我们生成了一套拥有自己 特色的成熟的IOS7适配方案。

3 现有的iOS7适配方案

客户端新增lua接口window:supportStatusBarInXML，是否通过模板来适配ios7状态栏效果。如果系统ios7,并且状态栏高度为0。 认为适配 ios7状态栏效果。且 window:supportStatusBarInXML返回值为 true或false, 数据类型为 boolen。

后台报文根据system:getInfo("version"）接口获取当前操作系统版本，如果为ios7或者以上版本， 则针对布局起始点为{0,0}编写body中第一个控件及其子控件样式，其他操作系统版本遵循以前规则。

由于适配IOS7导航栏，起始点为{0,0}，所以对于body的其他子节点如果写了top值需要客户端遍历修改top值为 (top先前值+增加导航栏高度)(客户端工作)。后台报文在写的时候尽量把header和下面的控件分别写为两个控件，保证body的子节点只有两个，便于客户端修改。(后台报文工作)。

4 举例

下面以两则例子详细说明

4.1 示例一：当导航栏为图片时：

用接口window:supportStatusBarInXML()，判断是否通过模板来适配ios7状态栏效果。 如果返回值为true，那么适配，即导航栏的height值和top值分别增加20px， 显示效果为导航栏的背景图片向上延伸至状态栏，使得状态栏的背景图片与导航栏的背景一致。

代码片段1:css 

<style>
    #{local ad_json= [[
    {
        "ad":
        [
            {
                "src":"ad3.jpg"
            },
            {
                "src":"ad4.jpg"
            }
        ]
    }
    ]]}#
    #{ local ad_lists = json:objectFromJSON(ad_json)}#
    #{ ad_lists = ad_lists["ad"]}#
    #{local ad_len = #ad_lists;}#

    #{
      local bar_flag = window:supportStatusBarInXML();
      local height = 160;
      local top = 6;
      if bar_flag then
          height = 180;
          top = 26;
      end;
    }#

    .div_bottom{bottom: 10px; width: 320px;}
    .div_btn_first{background-color: #147EEC;inline-block:inline;width: 8px;height: 8px;border-radius: 4px;border: 1px solid #147EEC;}
    .div_btn_first_1{background-color: #FFFFFF;border: 1px solid #FFFFFF;}
    .select_city{width: 40px;font-size: 18px;color: #FFFFFF;text-align: center;}
    .select_div{width: 20px;height: 30px;inline-block:inline;}
    .img_title{width: 20px;height: 10px;}
    .text_search{left:70px;border-radius: 10px;border: 1px solid
                 #FFFFFF;filter:progid(alpha='0.5');font-size: 12px; color: #FFFFFF;height: 30px;width: 160px;}
    .btn_alert{background-image: url(more.png); width: 20px;height: 20px;right: 10px;}

    .div_div_title{width: 305px;left: 15px;top:#{= top}#px;background-color:#FF0000;}
    .div_first_title{top:0px;background-image: url(ad3.jpg);width: 320px;height: #{= height}#px;}
    .div_header{top:0px;height: #{= height}#px;}
    #{for key,ad in pairs(ad_lists) do}#
      #{local left = ((key-1) * 320).."px";}#
      .div_display_title_#{= key}#{inline-block:inline;left:#{= left}#;background-image: url(#{= ad["src"]}#);}
    #{end}#
</style>

4.2 示例二：当导航栏为自定义的背景条时：

用接口window:supportStatusBarInXML()，判断是否通过模板来适配ios7状态栏效果。 如果返回值为true，那么适配，然后模板人员可以自定义状态栏背景色为任何合适的颜色。 本示例将状态栏的背景色设置为background-color: #FF0000。

代码片段1:css 

<style>
  .div_header_distance{height: 20px;background-color: #FF0000;}
  .div_positon{position: fixed;}
  .ert_div_title{width: 320px; height: 50px; background-color: #147EEC;}
  .ert_div_back {left:10px;width: 50px;height: 45px;inline-block:inline;}
  .ert_btn_title{width: 30px;height: 45px;font-size: 14px;color: #FFFFFF;}
  .ert_img_title{left:0px;width:8px; height: 14px;}
  .ert_btn_left{left:18px;}
  .ert_lbl_title{font-size: 18px;color: #FFFFFF;text-align: center;left:70px;width: 180px;}
  .ert_btn_right{right:10px;}
</style>

<body>
