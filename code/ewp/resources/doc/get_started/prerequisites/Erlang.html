1 简介

Erlang 是一门函数式编程语言（Functional Programming Language）。 这就是所有让人感到害怕――并让新的使用者望而却步的东西。然而有意思的是，Erlang 其实是一门非常小而且简单的语言。 可能你正在对Erlang 代码看上去的样子感到迷惑不解。

什么是模式匹配： 在Erlang中，= 表示一个模式匹配操作。 Erlang 大量使用了模式匹配语法；

这里有一个Erlang 代码的小例子（摘自新版的Programming Erlang 一书）：

-module(geometry).
-export([area/1]).
area({rectangle, Width, Ht}) -> Width * Ht;
area({square, X}) -> X * X;
area({circle, R}) -> 3.14159 * R * R.

现在，让我们在Erlang shell 中编译并运行它：

1> c(geometry).
{ok,geometry}
2> geometry:area({rectangle, 10, 5}).
50
3> geometry:area({circle, 1.4}).
6.15752

相当的简单…

以下是做类似事情的Java 代码：

abstract class Shape {
abstract double area();
}

class Circle extends Shape {
final double radius;
Circle(double radius) { this.radius = radius; }
double area() 
{ return Math.PI * radius*radius; }
}
class Rectangle extends Shape {
final double ht;
final double width;
Rectangle(double width, double height) {
this.ht = height;
this.width = width;
}
double area() { return width * ht; }
}

class Square extends Shape {
final double side;
Square(double side) {
this.side = side;
}
double area() { return side * side; }
}

2 基础语法

2.1 变量

变量首字母必须是大写。 Erlang的变量是单一赋值变量。

-module(tut1).
-export([fac/1, mult/2]).
fac(1) ->
    1;
fac(N) ->
    N * fac(N - 1).
mult(X, Y) ->
    X * Y.

在文件中的第一行，就告诉我们模块的名称。 说明tut1模块包含一个名称为fac的函数，并且带有一个参数(在我们的例子中为N)和一个名称为mult的函数，并且带有两个参数。而且这两个函数可以在tut1模块以外被调用。

fac函数第一部分说明1的阶乘是1,注意我们以一个;结束这一部分，这说明这个函数没有结束。 fac函数第二部分fac(N) ->N * fac(N - 1). 说明N的阶乘是，N与N - 1的阶乘的乘积(N! = N * (N - 1)!)。注意这部分以.结束，以说明本函数没有其它部分了。 在函数功能代码中的参数N, X, Y被叫做变量。

2.2 元子

元子是在Erlang中的另一个数据类型。 元子以小写字母开头， 例如：charles, centimeter,inch。元子只是一个简单的名字，其它什么都不是。他们不像变量可以带有一个值。

2.3 元组

元组以{和}括起来的。 所以我们可以写{inch, 3}来表示3英寸，并以{centimeter, 5}来表示5厘米。 一组元组有一个固定的大小。我们称元组中的东西为元素。所以在元组{moscow,{c,-10}}中，元素1是moscow，元素2是{c, -10}。

2.4 列表

列表在Erlang中被括在[和]里。 如一个不同城市温度的列表可能是这样的：

[1,2,3,4]
[{moscow, {c, -10}},
 {cape_town, {f, 70}}, 
 {stockholm, {c, -4}},
 {paris, {f, 28}}, 
 {london, {f, 36}}
]。

3 控制语句

3.1 if

if
    Condition 1 ->
    Action 1;
    Condition 2 ->
    Action 2;
    Condition 3 ->
    Action 3;
    Condition 4 ->
    Action 4;
    true ->
    Action 5
end

注意，在结尾没有;！ 条件(Conditions)和守卫一样，测试成功或是失败。Erlang会从最顶上开始向下，直到找到一个成功的条件为止，并执行此行条件下面的动作，并且忽略掉其后面其它的条件。如果没有条件匹配，会发生一个运行时(run-time)错误。元子true在条件中表示真，它常常被放在条件层，表示如果没有匹配的条件的话，应该做什么动作。

if小例子：

-module(tut9).
-export([test_if/2]).
test_if(A, B) ->
    if
        A == 5 ->
            io:format("A = 5~n", []),
            a_equals_5;
        B == 6 ->
            io:format("B = 6~n", []),
            b_equals_6;
        A == 2, B == 3 -> %i.e. A equals 2 and B equals 3
            io:format("A == 2, B == 3~n", []),
            a_equals_2_b_equals_3;
        A == 1 ; B == 7 -> %i.e. A equals 1 or B equals 7
            io:format("A == 1 ; B == 7~n", []),
            a_equals_1_or_b_equals_7
    end.

运行。

3.2 case

case是另一个Erlang结构。

-module(tut10).
-export([convert_length/1]).
convert_length(Length) ->
    case Length of
        {centimeter, X} ->
            {inch, X / 2.54};
        {inch, Y} ->
            {centimeter, Y * 2.54}
    end.

注意: case和if都有返回值，在上面的例子中case返回{inch,X/2.54}或 {centimeter,Y*2.54}。case的行为也可以使用守卫来代替。

小例子：

-module(tut11).
-export([month_length/2]).
month_length(Year, Month) ->
%% All years divisible by 400 are leap
%% Years divisible by 100 are not leap (except the 400 rule above)
%% Years divisible by 4 are leap (except the 100 rule above)
%%四年一闰，百年不闰，四百年再闰。
Leap = if
    trunc(Year / 400) * 400 == Year ->
        leap;
    trunc(Year / 100) * 100 == Year ->
        not_leap;
    trunc(Year / 4) * 4 == Year ->
        leap;
    true ->
        not_leap
    end,
    case Month of
        sep -> 30;
        apr -> 30;
        jun -> 30;
        nov -> 30;
        feb when Leap == leap -> 29;
        feb -> 28;
        jan -> 31;
        mar -> 31;
        may -> 31;
        jul -> 31;
        aug -> 31;
        oct -> 31;
        dec -> 31
    end.

4 简单例子

%% This module is in file tut5.erl
-module(tut5).
-export([format_temps/1]).
%% Only this function is exported

format_temps([])-> % No output for an empty list
    ok;
format_temps([City | Rest]) ->
    print_temp(convert_to_celsius(City)),
    format_temps(Rest).

convert_to_celsius({Name, {c, Temp}}) -> % No conversion needed
    {Name, {c, Temp}};
convert_to_celsius({Name, {f, Temp}}) -> % Do the conversion
    {Name, {c, (Temp - 32) * 5 / 9}}.

print_temp({Name, {c, Temp}}) ->
    io:format("~-15w ~w c~n", [Name, Temp]).

tut5:format_temps([{moscow, {c, -10}}, {cape_town, {f, 70}},{stockholm, {c, -4}}, {paris, {f, 28}}, {london, {f, 36}}]).
