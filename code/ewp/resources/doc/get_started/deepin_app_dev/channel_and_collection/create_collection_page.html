主菜单界面

  • 菜单界面开发
  • 生成菜单数据

1 说明

在创建了collection和channel后，就需要将这些菜单从EWP中获取然后展现在界面上。

2 前提

EWP提供了channel/list接口用于获取collection和channel菜单。

channel/list提供了两个回调用于在helloapp_channel中实现项目特定的逻辑。

这两个回调分别为：role_auth/2 和 list_collection_items/2。这两个回调具体说明请参照接口介绍章节。

3 菜单界面开发

  • 首先确定界面所需要json数据。 

    local menu_json = 
[[
{
  "return": {
      "error_code": "000000",
      "error_msg": "",
      "collection_list": [
          {
              "collection": "helloapp_account_manage",
              "name": "账户管理"
          },
          {
              "collection": "helloapp_tranfer_account",
              "name": "转账汇款"
          },
          {
              "collection": "helloapp_credit_card",
              "name": "信用卡"
          }
      ]
  }
}
]];

  • 根据json数据将菜单编写入欢迎界面login_alert.xhtml。 

      local menu_tab = json:objectFromJSON(menu_json);
  local collection_tab = menu_tab["return"]["collection_list"];

  local input = "";
  for key,collection in pairs(collection_tab) do
      local collection_id = collection["collection"];
      local collection_name = collection["name"];

      input = input .. 
      [[<input type="button" class="button_menu" name="]]..collection_id..[[" value="]]..collection_name..[[" onclick="post_channel(']]..collection_id..[[')"/>]];
  end;

  local div_menu = [[
  <div class="div_menu" name="div_menu" border="0">
  ]] .. input .. [[</div>]];

  local div_menu_ctrl = document:getElementsByName("div_menu");
  div_menu_ctrl[1]:setInnerHTML(div_menu);

  • 运行得到界面效果

4 生成菜单数据

根据界面所需的json数据我们需要在helloapp_channel的list_collection_items方法中实现返回的json数据。

在此方法中具体需要关注的为：CsKey, Input。

CsKey cs模板名称，在接口collection/list中为helloapp_list_i。其中helloapp为当前创建app的名称，i为客户端类型，i表示为高级客户端，比如android，iphone，winphone等。

Input 传入cs模板的tuplist参数。一般为此种格式：

[
{collection,[{id, "helloapp_account_manage"},
{name, [232,180,166,230,136,183,231,174,161,231,144,134]},
{menu_img, "undefined.png"},
{url, "http://192.168.241.3:4002/channel/list?id=helloapp_account_manage"}]},

{collection,[{id, "helloapp_tranfer_account"},
{name, [232,189,172,232,180,166,230,177,135,230,172,190]},
{menu_img, "undefined.png"},
{url, "http://192.168.241.3:4002/channel/list?id=helloapp_tranfer_account"}]},

{collection,[{id, "helloapp_credit_card"},
{name, [228,191,161,231,148,168,229,141,161]},
{menu_img, "undefined.png"},
{url, "http://192.168.241.3:4002/channel/list?id=helloapp_credit_card"}]}
]

创建helloapp_list_i.cs文件

#{cs set:count = 0}#
#{cs each:c=collection}#
#{cs    set:count = count+1}#
#{cs /each}#
{
"return": {
"error_code": "000000",
"error_msg": "",
"collection_list": [
#{cs set:index = 0}#
#{cs each:c=collection}#
#{cs    set:index = index +1}#
{
"collection": "#{cs var:c.id}#",
"name": "#{cs var:c.name}#"
#{cs    if:index == count}#
}
#{cs    else}#
},
#{cs    /if}#
#{cs /each}#
]
}
}

修改界面中json数据为从channel/list接口获取,由于EWP目前提供的route中channel/list对应的接口名称为app_s/list，所以请求为app_s/list。

修改欢迎界面中数据从全局table中获取。

login_alert.xhtml: local menu_json = global_tab["helloapp_training"];

修改登录界面为登陆成功后请求菜单然后存入global_tab["helloapp_training"]中。

    local function list_callback(result)
        local responseCode = result["responseCode"];

        if responseCode == 200 then
            global_tab["helloapp_training"] = result["responseBody"];

            --请求欢迎页面
            http:postAsyn(nil,"test_s/get_page","name=training/login_alert.xhtml",qry_page_callback,nil);
        else
            window:alert("服务器请求出错！");
        end;

    end;

    --在login方法中修改请求欢迎界面为请求菜单。
    http:postAsyn(nil,"app_s/list","id=helloapp_training&type=collections",list_callback,nil);
