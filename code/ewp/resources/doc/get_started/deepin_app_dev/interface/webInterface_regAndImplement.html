Web接口规范说明

EWP对Web接口定义了如下规范：

  1. 每个Web接口需要定义一个唯一的URL入口，格式为Host:Port/Component/Function。其中Host和Port为App服务的访问主机和端口，Component和Function为接口所属组件和功能，这两个值在Web接口注册时指定。
  2. 每个Web接口需要定义一个处理模块及函数（参数个数须为1），模块名须为XXX_controller.erl。处理函数须在处理模块中导出。
  3. App需要设置.app描述文件的controllers配置，描述URL与处理模块和函数的映射关系。通过已配置的映射关系，EWP可以将访问指定URL的请求分发到对应的处理函数，并根据返回值生成响应。

注册Web接口

注册Web接口实际上就是在.app描述文件的controllers配置中添加一个URL与处理模块和函数的映射关系。一个映射关系的配置格式为{URLSpec, ControllerSpec, Options}，含义如下：

  1. URLSpec可以为{Component, Function}或Component，表示将访问URL为Component/Function或Component/Any的请求分发到指定处理模块和函数，Any表示任意的Function。Component和Function类型均为string。
  2. ControllerSpec可以为{Module, HandleFunc}或Module，表示将访问指定URL的请求分发到Module:HandleFunc或Module:Function函数处理，Function为URL对应的值（如URL为ebank/test，则Function为test）。Module和HandleFunc类型均为atom。
  3. Options为[{decrypt, boolean()}, {verify, boolean()}]，表示加解密和验签开关。打开时，所有请求需要在成功建立EMP安全信道之后才可以访问指定的URL。

Web接口实现

注册Web接口之后，我们还要实现对应的处理模块和函数。继续上面的例子，接下来我们要编写demo_controller模块并实现handle_hw/1函数。在App的src/controllers目录下创建demo_controller.erl文件，添加如下代码：

-module(demo_controller).
-include("ewp.hrl").
-export([handle_hw/1]).

handle_hw(Arg) ->
    ewp_params:save_yaws_arg(Arg),
    Input = ?param("input", ""),
    ewp_render_util:render_html("Hello World!!!<br/>Your input is " ++ Input, []).

handle_hw的参数Arg是一个描述当前请求内容的数据结构，函数体的前两行描述了如何通过Arg来获取请求参数。执行ewp_params:save_yaws_arg(Arg)之后，请求参数将被保存起来，通过?param宏便可以获取需要的请求参数。需要注意的是，使用?param宏前需要引入ewp.hrl文件。

handle_hw的返回值需要使用ewp_render_util提供的Api来生成，可以参考EWP的Api文档来了解该模块具体的Api规范。例子中使用render_html/2来生成了一个简单的html页面。
