插件规格

每个Callback插件都有其应用场景、回调接口规格以及回调函数调用顺序，有些插件还会有一些Web接口作为程序入口。在选择或开发插件之前，我们需要了解插件的具体规格。以下为user插件的规格：

  • 程序入口：以user_controller:login/1、user_controller:logout/1和user_controller:main_page/1为入口，App可以通过设置.app文件的controllers配置将指定的Web接口与这三个函数映射起来。在ewp.conf中，默认配置了一些映射关系（如下图），App也可以使用这些默认接口。
  • 应用场景：user插件主要用于处理用户发送的登录和登出请求。
  • 回调接口规格：
    • auth(Arg) -> {ok, Data :: term()} | {error, Reason :: string()}
      用来实现登录验证流程，验证成功须返回{ok, Data}，失败须返回{error, Reason}。Data会作为gen_login_page/2接口的输入参数。Reason会作为错误信息直接抛出。
    • gen_login_page(Output :: atom(), Data :: term()) -> binary() | string() 用来生成登录成功的页面，返回值类型为string或binary。Output为客户端指定的o参数，该参数多用来区分EMP客户端类型。Data为登录验证成功时，auth/1的返回内容。
    • logout({SessionId :: string(), Session :: term()}) -> any()
      用来实现登出流程。输入参数是session id和session内容组成的tuple。可返回任意值。
    • main_page(Arg) -> {response, term()} 该接口很少用到，本文不做详细介绍。
  • 回调接口调用流程：user插件回调接口调用流程如下图所示。
    登录流程会以user_controller:login/1为入口，首先调用auth/1回调接口然后再调用gen_login_page/2回调接口。登出流程会以user_controller:logout/1为入口，然后调用logout/1回调接口。

定制插件

了解插件的规格之后，我们就可以通过编写插件回调接口来定制插件的功能。我们以实现简单的登录流程为目标，来简要说明插件开发的流程。步骤如下：

  1. 修改.app文件，将user添加到plugins列表中（如下图）。这样做可以在App启动时检查回调接口是否正确声明，避免运行时出错。

  2. 在App的src/plugins目录下创建AppName_user.erl模块，其中AppName为App名称（如ebank）。添加下列代码： ``` -module(ebank_user).

-behavior(generic_user).

-include("ewp.hrl").

-export([main_page/1, auth/1, gen_login_page/2, logout/1]).

auth(Arg) -> Name = case ?param("user_id") of "1" -> "David"; "2" -> "Jones"; -> undefined end, {ok, Name}.

gen_login_page(_OutputFormat, undefined) -> "Invalid user"; gen_login_page(_OutputFormat, Name) -> "" ++ Name ++ "".

logout(_) -> undefined.

mainpage() -> undefined. ```

在3.3中，我们注册了一个Web接口并将其与user_controller:login/1映射起来，这样通过我们注册的Web接口，我们便可以使用user插件提供的登录功能。实际开发中，这个步骤不是必须的，我们可以在建立EMP信道后使用默认的Web接口。
