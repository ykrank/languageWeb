EMP用户手册

Session共享操作

1 概述

本文旨在通过session优化解决了会话保持错误和数据不一致问题后，介绍EWP session的正常操作流程。

1.1 准备工作

如果想使用session共享功能，必须要配置两个节点以上的集群环境，即需要两台或者更多的服务器。同时EWP需要5.2.60-R(不包含)以后的版本。

2 配置

2.1 主机名配置

在配置集群环境的时候，可能会遇到两台服务器主机名相同的情况，这样，会造成管理的混乱和其他问题，下面介绍修改主机名的方法。 例如，Centos系统下，有两台主机名相同的服务器"localhost"，使用shell命令sudo vi /etc/sysconfig/network，会有如下配置内容：

NETWORKING=yes
# HOSTNAME 主机名配置项
HOSTNAME=localhost

变更为：

主节点配置：
NETWORKING=yes
HOSTNAME=ewp_master

从节点配置：
NETWORKING=yes
HOSTNAME=ewp_slave

然后需要配置主机名与IP的映射关系，文件路径/etc/hosts，"主机名称"通过shell命令hostname查询，使用shell命令sudo vi /etc/hosts进行编辑。

主节点配置内容如下：
#网络IP地址     主机名或域名      主机名别名
127.0.0.1     localhost
127.0.0.1     ewp_master
192.168.0.1   ewp_master
192.168.0.2   ewp_slave
...

从节点配置内容如下：
#网络IP地址     主机名或域名      主机名别名
127.0.0.1     localhost
127.0.0.1     ewp_slave
192.168.0.1   ewp_master
192.168.0.2   ewp_slave
...

配置完毕后，需要执行shell命令reboot，重启服务器，完成主机名的变更。

2.2 ewp.conf

ewp.conf文件路径 APP/config/ewp.conf 在集群环境中，只有一个主节点，其余全为从节点，所以ewp.conf配置会有些不同。 注：开启session共享功能后，会降低session操作的效率，请结合项目实际情况选择是否启用此功能。

主节点配置：
...
%% Mnesia and database node type.
{db_node_type, master}.
%% (Only Master support disc storage).
{mnesia, [{mnesia_dir, "./bin/mnesia"}]}.
%% 是否启用session共享功能：true | false。默认值 true
%% {session_shared, true}.
...

从节点配置：
...
%% Salve Config Sample
{db_node_type, slave}.
%% {master, 主节点Node}.
{master, ebank@ewp_master}.
%% 拷贝主节点'ram_copies'类型的mnesia数据库，默认值是全部。
%% {mnesia_table_copy, [ota_phones, users]}.
%% 是否启用session共享功能：true | false。默认值 true
%% {session_shared, true}.
...

2.3 使用

session的API封装在session_service模块，优化后，并没有改变API的参数和返回值，所以使用方式与优化前一样。

2.4 启动

集群环境的启动，需要注意节点的启动顺序，先启动主节点，之后再启动从节点。

3 交付反馈问题汇总

3.1 从节点启动失败

问题描述：从节点启动报错。

 =INFO REPORT==== 31-Mar-2015::12:33:13 ===
ewp_mnesia:196 call master node ebank@ewp_master to add slave and share the schema

=ERROR REPORT==== 31-Mar-2015::12:33:13 ===
ewp_mnesia:202 error: master failed to add slave for reason :{badrpc,nodedown}

报错信息为{badrpc, nodedown}，原因是主、从两个节点之间不能正常连接，所以需要确认：

  • 1、从机ewp.conf中，{master, MasterNode}. MasterNode是否与主机节点名称一致。
  • 2、参照《EMP用户手册》10.2 Session共享操作 2.1主机名配置，配置主机名与IP的映射关系。
  • 3、在从机通过测试erlang shellerl -sname test -setcookie ewpcool，ping主节点net_adm:ping(MasterNode).，如果返回值为pong，说明可以连接，启动从节点。如果返回pang，说明连接失败，说明有防火墙存在，继续参照第4步。
  • 4、由于EWP在启动时，设置了端口范围4100 ~ 4110，所以需要打开防火墙端口4100 ~ 4110，在防火墙配置中添加 -A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 4100:4110 -j ACCEPT。还可以设置接收来自某一个ip地址(192.168.65.71主节点IP)的所有input-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp -s 192.168.65.71 -j ACCEPT，则不需要再设置端口。

防火墙配置(Centos)路径：vi /etc/sysconfig/iptables，下面是配置范例

# Firewall configuration written by system-config-securitylevel
# Manual customization of this file is not recommended.
*filter
:INPUT ACCEPT [0:0]
:FORWARD ACCEPT [0:0]
:OUTPUT ACCEPT [0:0]
:RH-Firewall-1-INPUT - [0:0]
-A INPUT -j RH-Firewall-1-INPUT
-A FORWARD -j RH-Firewall-1-INPUT
-A RH-Firewall-1-INPUT -i lo -j ACCEPT
-A RH-Firewall-1-INPUT -p icmp --icmp-type any -j ACCEPT
-A RH-Firewall-1-INPUT -p 50 -j ACCEPT
-A RH-Firewall-1-INPUT -p 51 -j ACCEPT
# 允许已建立的或相关连的通行
-A RH-Firewall-1-INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# 允许访问端口
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 53 -j ACCEPT
-A RH-Firewall-1-INPUT -m state --state NEW -m udp -p udp --dport 53 -j ACCEPT
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 22 -j ACCEPT
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 25 -j ACCEPT
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 80 -j ACCEPT
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 443 -j ACCEPT

# 允许访问EWP端口
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp --dport 4100:4110 -j ACCEPT

# 允许访问ip所有端口
-A RH-Firewall-1-INPUT -m state --state NEW -m tcp -p tcp -s 192.168.65.71 -j ACCEPT

-A RH-Firewall-1-INPUT -j REJECT --reject-with icmp-host-prohibited
COMMIT

3.2 EWP集群搭建建议

3.2.1 master机器不要做业务节点，部署在内网，用作管理后台的管理操作。

因为所有从节点的离线资源会以主节点为准，在从节点启动时同步到本地。当主节点坏掉后，从节点更新了离线资源，在启动主节点时，不会更新离线资源，这样在以后的离线资源同步中，会造成数据错误或丢失。
